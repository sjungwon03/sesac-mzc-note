# 12. 예외 처리

## 예외란?

예외는 프로그램 실행 중 발생할 수 있는 비정상 상황이다.

```java
int result = 10 / 0; // ArithmeticException
```

예외를 처리하지 않으면 프로그램이 비정상 종료될 수 있다.

---

## 예외 계층

```text
Throwable
 ├─ Error
 └─ Exception
     ├─ RuntimeException
     └─ 기타 Exception
```

| 구분 | 설명 |
| --- | --- |
| Error | 시스템 수준의 심각한 문제, 일반적으로 처리 대상이 아님 |
| Exception | 프로그램에서 처리 가능한 예외 |
| RuntimeException | 실행 중 발생하는 예외, 컴파일러가 처리 강제하지 않음 |

---

## try-catch-finally

```java
try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    System.out.println("0으로 나눌 수 없습니다.");
} finally {
    System.out.println("항상 실행");
}
```

| 블록 | 역할 |
| --- | --- |
| `try` | 예외가 발생할 수 있는 코드 |
| `catch` | 예외 처리 코드 |
| `finally` | 예외 발생 여부와 관계없이 실행 |

---

## checked exception과 unchecked exception

| 구분 | checked exception | unchecked exception |
| --- | --- | --- |
| 처리 강제 | 컴파일러가 강제 | 강제하지 않음 |
| 대표 예 | `IOException`, `SQLException` | `NullPointerException`, `ArithmeticException` |
| 발생 시점 | 컴파일 단계에서 확인 가능 | 실행 중 발생 |

checked exception은 `try-catch`로 처리하거나 `throws`로 호출한 쪽에 넘겨야 한다.

---

## throws

메서드 내부에서 예외를 직접 처리하지 않고 호출한 쪽으로 넘길 때 사용한다.

```java
void readFile() throws IOException {
    // 파일 읽기 코드
}
```

---

## 사용자 정의 예외

업무 규칙에 맞는 예외를 직접 만들 수 있다.

```java
class InvalidAgeException extends Exception {
    InvalidAgeException(String message) {
        super(message);
    }
}
```

---

## 핵심 정리

- 예외는 실행 중 발생할 수 있는 비정상 상황이다.
- `try-catch`는 예외를 처리하고 프로그램 종료를 막는다.
- `finally`는 자원 정리 코드에 주로 사용한다.
- checked exception은 반드시 처리하거나 `throws`로 넘겨야 한다.
