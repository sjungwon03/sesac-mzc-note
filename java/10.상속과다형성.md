# 10. 상속과 다형성

## 상속

상속은 기존 클래스의 필드와 메서드를 새로운 클래스가 물려받아 재사용하는 기능이다.

```java
class Animal {
    void eat() {
        System.out.println("먹는다");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("멍멍");
    }
}
```

용어:

- 부모 클래스: 상속해 주는 클래스
- 자식 클래스: 상속받는 클래스
- `extends`: 상속을 표현하는 키워드

Java는 클래스의 다중 상속을 허용하지 않는다.

---

## super

`super`는 부모 객체를 가리킨다.

```java
class Parent {
    String name = "parent";
}

class Child extends Parent {
    String name = "child";

    void print() {
        System.out.println(this.name);
        System.out.println(super.name);
    }
}
```

`super()`는 부모 생성자를 호출한다. 생성자 첫 줄에서만 사용할 수 있다.

---

## 오버라이딩

오버라이딩은 부모 메서드를 자식 클래스에서 재정의하는 것이다.

```java
class Animal {
    void sound() {
        System.out.println("소리");
    }
}

class Cat extends Animal {
    @Override
    void sound() {
        System.out.println("야옹");
    }
}
```

오버라이딩 조건:

- 메서드 이름이 같아야 한다.
- 매개변수 목록이 같아야 한다.
- 반환 타입이 같거나 호환되어야 한다.
- 접근 제한자는 부모보다 좁게 변경할 수 없다.

---

## 다형성

다형성은 하나의 부모 타입 변수가 여러 자식 객체를 참조할 수 있는 성질이다.

```java
Animal animal = new Cat();
animal.sound(); // Cat의 sound 실행
```

부모 타입으로 자식 객체를 다룰 수 있어 코드 확장성이 좋아진다.

---

## 타입 변환

### 자동 타입 변환

자식 객체는 부모 타입으로 자동 변환된다.

```java
Dog dog = new Dog();
Animal animal = dog;
```

### 강제 타입 변환

부모 타입으로 참조 중인 객체를 다시 자식 타입으로 사용할 때 필요하다.

```java
if (animal instanceof Dog) {
    Dog d = (Dog) animal;
    d.bark();
}
```

---

## 핵심 정리

- 상속은 공통 기능을 부모 클래스로 분리해 재사용한다.
- `super`는 부모 멤버 또는 부모 생성자 접근에 사용한다.
- 오버라이딩은 부모 메서드를 자식 클래스에 맞게 재정의하는 것이다.
- 다형성은 부모 타입으로 여러 자식 객체를 다루는 방식이다.
