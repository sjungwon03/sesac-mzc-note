# 17. 멀티스레드와 네트워크

## 프로세스와 스레드

| 구분 | 설명 |
| --- | --- |
| 프로세스 | 실행 중인 프로그램 |
| 스레드 | 프로세스 안에서 실행되는 작업 단위 |

하나의 프로세스는 여러 스레드를 가질 수 있다. 여러 작업을 동시에 처리해야 할 때 멀티스레드를 사용한다.

---

## 스레드 생성

### Thread 상속

```java
class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("thread run");
    }
}

new MyThread().start();
```

### Runnable 구현

```java
Runnable task = () -> System.out.println("task run");
Thread thread = new Thread(task);
thread.start();
```

`run()`을 직접 호출하면 새 스레드가 만들어지지 않는다. 반드시 `start()`를 호출해야 한다.

---

## 동기화

여러 스레드가 같은 데이터를 동시에 수정하면 예상하지 못한 결과가 발생할 수 있다.

```java
synchronized void increase() {
    count++;
}
```

`synchronized`는 한 번에 하나의 스레드만 해당 코드에 접근하게 한다.

---

## 네트워크 기본 개념

| 용어 | 설명 |
| --- | --- |
| IP | 네트워크에서 장치를 구분하는 주소 |
| Port | 하나의 장치 안에서 프로그램을 구분하는 번호 |
| TCP | 연결 기반의 신뢰성 있는 통신 방식 |
| Socket | 네트워크 통신의 양 끝점 |

---

## TCP 통신 흐름

서버:

```java
ServerSocket serverSocket = new ServerSocket(5000);
Socket socket = serverSocket.accept();
```

클라이언트:

```java
Socket socket = new Socket("localhost", 5000);
```

통신 흐름:

```text
서버 소켓 생성 → 클라이언트 연결 요청 → 서버 accept → Socket으로 데이터 송수신
```

---

## 핵심 정리

- 프로세스는 실행 중인 프로그램, 스레드는 작업 실행 단위이다.
- Java 스레드는 `Thread` 또는 `Runnable`로 만들 수 있다.
- 새 스레드를 실행하려면 `start()`를 호출해야 한다.
- 공유 데이터는 동기화가 필요할 수 있다.
- TCP 통신은 `ServerSocket`과 `Socket`을 사용한다.
