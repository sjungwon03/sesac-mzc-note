# 13. 컬렉션 프레임워크

## 컬렉션 프레임워크란?

컬렉션 프레임워크는 여러 데이터를 효율적으로 저장하고 처리하기 위한 Java 표준 자료구조 모음이다.

배열과 달리 대부분의 컬렉션은 크기가 동적으로 변경된다.

---

## 주요 인터페이스

| 인터페이스 | 특징 | 대표 구현체 |
| --- | --- | --- |
| `List` | 순서 있음, 중복 허용 | `ArrayList`, `LinkedList` |
| `Set` | 순서 없음, 중복 불가 | `HashSet`, `TreeSet` |
| `Map` | key-value 구조, key 중복 불가 | `HashMap`, `TreeMap` |

---

## List

순서가 중요하고 중복 값을 허용해야 할 때 사용한다.

```java
List<String> names = new ArrayList<>();
names.add("Kim");
names.add("Lee");
names.add("Kim");

System.out.println(names.get(0));
```

주요 메서드:

| 메서드 | 설명 |
| --- | --- |
| `add(value)` | 값 추가 |
| `get(index)` | 인덱스로 조회 |
| `set(index, value)` | 값 변경 |
| `remove(index)` | 값 삭제 |
| `size()` | 저장 개수 확인 |

---

## Set

중복을 허용하지 않는 데이터 집합이 필요할 때 사용한다.

```java
Set<String> names = new HashSet<>();
names.add("Kim");
names.add("Kim");

System.out.println(names.size()); // 1
```

객체의 중복 판단은 `equals()`와 `hashCode()` 결과에 영향을 받는다.

---

## Map

key와 value를 쌍으로 저장한다.

```java
Map<String, Integer> scores = new HashMap<>();
scores.put("Kim", 90);
scores.put("Lee", 80);

System.out.println(scores.get("Kim"));
```

주요 메서드:

| 메서드 | 설명 |
| --- | --- |
| `put(key, value)` | 값 저장 |
| `get(key)` | key로 값 조회 |
| `remove(key)` | key로 삭제 |
| `containsKey(key)` | key 존재 확인 |
| `keySet()` | key 목록 조회 |

---

## 제네릭

제네릭은 컬렉션에 저장할 타입을 미리 지정하는 문법이다.

```java
List<Integer> numbers = new ArrayList<>();
numbers.add(10);
```

장점:

- 잘못된 타입 저장을 컴파일 단계에서 방지한다.
- 값을 꺼낼 때 형 변환을 줄일 수 있다.

---

## 핵심 정리

- `List`는 순서와 중복이 필요할 때 사용한다.
- `Set`은 중복 제거가 필요할 때 사용한다.
- `Map`은 key로 value를 찾는 구조가 필요할 때 사용한다.
- 제네릭은 컬렉션에 저장할 타입을 명확히 제한한다.
