# 18. JDBC

## JDBC란?

JDBC는 Java에서 관계형 데이터베이스에 접속하고 SQL을 실행하기 위한 표준 API이다.

```text
Java Application
        ↓
JDBC API
        ↓
JDBC Driver
        ↓
Database
```

DB 종류가 달라도 Java 코드는 JDBC 표준 인터페이스를 중심으로 작성할 수 있다.

---

## JDBC 프로그래밍 흐름

| 단계 | 설명 | 주요 객체 |
| --- | --- | --- |
| 1 | 드라이버 로딩 | `Class.forName()` |
| 2 | DB 연결 | `Connection` |
| 3 | SQL 준비 | `PreparedStatement` |
| 4 | SQL 실행 | `executeQuery()`, `executeUpdate()` |
| 5 | 결과 처리 | `ResultSet` |
| 6 | 자원 해제 | `close()` |

---

## 주요 객체

### Connection

데이터베이스와의 연결을 나타낸다.

```java
Connection conn = DriverManager.getConnection(url, user, password);
```

### PreparedStatement

SQL을 미리 준비하고 파라미터를 안전하게 바인딩한다.

```java
String sql = "SELECT * FROM users WHERE id = ?";
PreparedStatement pstmt = conn.prepareStatement(sql);
pstmt.setInt(1, 1);
```

### ResultSet

`SELECT` 결과를 행 단위로 읽는다.

```java
ResultSet rs = pstmt.executeQuery();

while (rs.next()) {
    System.out.println(rs.getString("name"));
}
```

---

## SELECT 예시

```java
String sql = "SELECT id, name FROM users WHERE id = ?";

try (Connection conn = DriverManager.getConnection(url, user, password);
     PreparedStatement pstmt = conn.prepareStatement(sql)) {

    pstmt.setInt(1, 1);

    try (ResultSet rs = pstmt.executeQuery()) {
        while (rs.next()) {
            int id = rs.getInt("id");
            String name = rs.getString("name");
            System.out.println(id + ": " + name);
        }
    }
} catch (SQLException e) {
    e.printStackTrace();
}
```

---

## INSERT, UPDATE, DELETE

데이터 변경 SQL은 `executeUpdate()`를 사용한다.

```java
String sql = "INSERT INTO users(name, email) VALUES(?, ?)";

try (Connection conn = DriverManager.getConnection(url, user, password);
     PreparedStatement pstmt = conn.prepareStatement(sql)) {

    pstmt.setString(1, "Kim");
    pstmt.setString(2, "kim@example.com");

    int count = pstmt.executeUpdate();
    System.out.println(count + "행 처리");
} catch (SQLException e) {
    e.printStackTrace();
}
```

---

## PreparedStatement를 사용하는 이유

- SQL Injection 위험을 줄일 수 있다.
- SQL과 값을 분리해 코드가 읽기 쉽다.
- 같은 SQL을 여러 번 실행할 때 효율적이다.

---

## 핵심 정리

- JDBC는 Java와 RDBMS를 연결하는 표준 API이다.
- DB 연결은 `Connection`, SQL 실행은 `PreparedStatement`, 조회 결과는 `ResultSet`이 담당한다.
- 조회는 `executeQuery()`, 변경은 `executeUpdate()`를 사용한다.
- DB 자원은 반드시 닫아야 하며 `try-with-resources`를 사용하면 안전하다.
