# 6. 클래스와 객체

## 클래스와 객체

클래스는 객체를 만들기 위한 설계도이고, 객체는 클래스를 바탕으로 생성된 실제 대상이다.

```java
class Student {
    String name;
    int age;

    void introduce() {
        System.out.println(name + ", " + age);
    }
}
```

```java
Student student = new Student();
student.name = "Kim";
student.age = 20;
student.introduce();
```

| 용어 | 설명 |
| --- | --- |
| 클래스 | 객체를 만들기 위한 설계도 |
| 객체 | 클래스로부터 생성된 대상 |
| 인스턴스 | 메모리에 생성된 객체 |
| 필드 | 객체가 가지는 데이터 |
| 메서드 | 객체가 수행하는 기능 |

---

## 객체 생성

객체는 `new` 키워드로 생성한다.

```java
Student s1 = new Student();
Student s2 = new Student();
```

`s1`과 `s2`는 같은 클래스에서 만들어졌지만 서로 다른 객체이다.

---

## 필드

필드는 객체의 상태를 저장한다.

```java
class Car {
    String model;
    int speed;
}
```

필드에는 타입별 기본값이 들어간다.

| 타입 | 기본값 |
| --- | --- |
| 정수 | `0` |
| 실수 | `0.0` |
| boolean | `false` |
| 참조 타입 | `null` |

---

## 메서드

메서드는 객체의 동작을 정의한다.

```java
class Calculator {
    int add(int a, int b) {
        return a + b;
    }
}
```

메서드 구조:

```java
반환타입 메서드명(매개변수) {
    실행문;
    return 반환값;
}
```

반환값이 없으면 반환 타입으로 `void`를 사용한다.

---

## 객체 간 협력

객체는 자신의 필드와 메서드를 사용하고, 필요하면 다른 객체의 메서드를 호출해 협력한다.

```java
class Printer {
    void print(String message) {
        System.out.println(message);
    }
}

class User {
    void usePrinter() {
        Printer printer = new Printer();
        printer.print("Hello");
    }
}
```

---

## 주의사항

- 클래스명은 보통 대문자로 시작한다.
- 객체를 생성하지 않으면 인스턴스 필드와 메서드를 사용할 수 없다.
- 참조 변수가 `null`인 상태에서 멤버에 접근하면 `NullPointerException`이 발생한다.
- 하나의 파일에는 여러 클래스를 작성할 수 있지만 `public` 클래스는 하나만 둘 수 있다.

---

## 핵심 정리

- 클래스는 설계도, 객체는 실제 생성된 대상이다.
- 필드는 상태, 메서드는 동작을 표현한다.
- 객체는 `new`로 생성하고 참조 변수로 접근한다.
- 인스턴스마다 필드 값은 독립적으로 관리된다.
