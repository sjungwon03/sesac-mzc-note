# 8. 패키지와 접근 제한자

## 패키지

패키지는 관련 있는 클래스를 묶어 관리하는 이름 공간이다.

```java
package com.example.app;
```

역할:

- 클래스 이름 충돌 방지
- 기능별 코드 분류
- 접근 범위 관리

패키지 선언은 소스 파일의 가장 위에 작성한다.

---

## import

다른 패키지의 클래스를 사용할 때는 `import`를 사용한다.

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
    }
}
```

`java.lang` 패키지는 자동으로 import되므로 따로 작성하지 않아도 된다.

---

## 접근 제한자

접근 제한자는 클래스, 필드, 생성자, 메서드의 사용 범위를 제한한다.

| 제한자 | 같은 클래스 | 같은 패키지 | 자식 클래스 | 전체 |
| --- | --- | --- | --- | --- |
| `public` | 가능 | 가능 | 가능 | 가능 |
| `protected` | 가능 | 가능 | 가능 | 불가 |
| default | 가능 | 가능 | 불가 | 불가 |
| `private` | 가능 | 불가 | 불가 | 불가 |

default는 접근 제한자를 작성하지 않은 상태를 의미한다.

---

## 캡슐화와 private

필드는 보통 `private`으로 숨기고, 필요한 경우 getter/setter로 접근한다.

```java
class Member {
    private String name;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}
```

캡슐화를 사용하면 객체 내부 상태를 안전하게 관리할 수 있다.

---

## 클래스 접근 제한

일반 클래스에는 `public` 또는 default 접근만 사용할 수 있다.

```java
public class User {
}

class UserHelper {
}
```

`public` 클래스는 파일명과 클래스명이 같아야 한다.

---

## 핵심 정리

- 패키지는 클래스를 기능별로 묶고 이름 충돌을 방지한다.
- `import`는 다른 패키지의 클래스를 간단한 이름으로 사용하게 해준다.
- `private`은 같은 클래스 내부에서만 접근 가능하다.
- 필드는 `private`, 메서드는 필요한 범위만 `public`으로 여는 것이 기본 원칙이다.
