# 16. 입출력

## 스트림이란?

스트림은 프로그램과 외부 자원 사이에서 데이터가 이동하는 통로이다.

```text
입력 스트림: 외부 자원 → 프로그램
출력 스트림: 프로그램 → 외부 자원
```

---

## 바이트 스트림과 문자 스트림

| 구분 | 입력 | 출력 | 사용 대상 |
| --- | --- | --- | --- |
| 바이트 스트림 | `InputStream` | `OutputStream` | 이미지, 영상, 바이너리 파일 |
| 문자 스트림 | `Reader` | `Writer` | 텍스트 파일 |

텍스트 파일은 문자 스트림을 사용하는 것이 일반적이다.

---

## 파일 쓰기

```java
try (FileWriter writer = new FileWriter("sample.txt")) {
    writer.write("Hello Java");
} catch (IOException e) {
    e.printStackTrace();
}
```

`try-with-resources`를 사용하면 스트림을 자동으로 닫을 수 있다.

---

## 파일 읽기

```java
try (BufferedReader reader = new BufferedReader(new FileReader("sample.txt"))) {
    String line;
    while ((line = reader.readLine()) != null) {
        System.out.println(line);
    }
} catch (IOException e) {
    e.printStackTrace();
}
```

`BufferedReader`는 한 줄씩 읽을 수 있어 텍스트 처리에 자주 사용된다.

---

## 보조 스트림

보조 스트림은 기본 스트림에 기능을 추가한다.

| 보조 스트림 | 역할 |
| --- | --- |
| `BufferedInputStream` | 바이트 입력 성능 향상 |
| `BufferedOutputStream` | 바이트 출력 성능 향상 |
| `BufferedReader` | 문자 입력 성능 향상, 줄 단위 읽기 |
| `BufferedWriter` | 문자 출력 성능 향상 |

---

## 핵심 정리

- 스트림은 데이터 입출력 통로이다.
- 바이트 스트림은 모든 데이터, 문자 스트림은 텍스트 처리에 적합하다.
- 입출력 작업은 `IOException` 처리가 필요하다.
- 스트림은 사용 후 반드시 닫아야 하며, `try-with-resources`를 사용하면 안전하다.
