# 14. 함수형 프로그래밍

## 함수형 프로그래밍이란?

함수형 프로그래밍은 데이터를 변경하기보다 함수의 조합으로 결과를 만들어 내는 방식이다.

Java에서는 람다식, 함수형 인터페이스, Stream API를 통해 함수형 스타일을 지원한다.

---

## 람다식

람다식은 익명 함수를 간결하게 표현하는 문법이다.

```java
(매개변수) -> 실행문
```

예시:

```java
Runnable runnable = () -> System.out.println("run");
```

매개변수가 하나면 괄호를 생략할 수 있다.

```java
Consumer<String> printer = message -> System.out.println(message);
```

---

## 함수형 인터페이스

함수형 인터페이스는 추상 메서드를 하나만 가진 인터페이스이다.

```java
@FunctionalInterface
interface Calculator {
    int calculate(int a, int b);
}
```

```java
Calculator add = (a, b) -> a + b;
System.out.println(add.calculate(1, 2));
```

`@FunctionalInterface`를 붙이면 추상 메서드가 2개 이상일 때 컴파일 오류로 확인할 수 있다.

---

## 주요 함수형 인터페이스

| 인터페이스 | 역할 | 메서드 |
| --- | --- | --- |
| `Consumer<T>` | 값을 받아 소비 | `accept` |
| `Supplier<T>` | 값을 공급 | `get` |
| `Function<T, R>` | 값을 변환 | `apply` |
| `Predicate<T>` | 조건 검사 | `test` |

---

## Stream API

Stream은 컬렉션 데이터를 선언형으로 처리하는 API이다.

```java
List<String> names = List.of("Kim", "Lee", "Park");

names.stream()
    .filter(name -> name.length() >= 3)
    .map(String::toUpperCase)
    .forEach(System.out::println);
```

처리 흐름:

```text
스트림 생성 → 중간 연산 → 최종 연산
```

| 구분 | 예시 | 설명 |
| --- | --- | --- |
| 중간 연산 | `filter`, `map`, `sorted` | 결과로 Stream 반환 |
| 최종 연산 | `forEach`, `collect`, `count` | 실제 연산 실행 |

---

## 핵심 정리

- 람다식은 익명 구현을 간결하게 표현한다.
- 함수형 인터페이스는 추상 메서드가 하나만 있어야 한다.
- Stream은 원본 데이터를 변경하지 않고 처리 결과를 만든다.
- 중간 연산은 지연 실행되고, 최종 연산이 호출될 때 실제 처리된다.
