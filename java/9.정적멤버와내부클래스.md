# 9. 정적 멤버와 내부 클래스

## static 멤버

`static` 멤버는 객체마다 따로 존재하지 않고 클래스에 소속된다.

```java
class Calculator {
    static double pi = 3.14159;

    static int add(int a, int b) {
        return a + b;
    }
}
```

사용 예시:

```java
System.out.println(Calculator.pi);
System.out.println(Calculator.add(1, 2));
```

객체를 만들지 않고 클래스 이름으로 접근한다.

---

## 인스턴스 멤버와 static 멤버

| 구분 | 인스턴스 멤버 | static 멤버 |
| --- | --- | --- |
| 소속 | 객체 | 클래스 |
| 생성 시점 | 객체 생성 시 | 클래스 로딩 시 |
| 접근 방법 | 참조 변수 | 클래스 이름 |
| 데이터 공유 | 객체마다 별도 | 모든 객체가 공유 |

```java
class Counter {
    int instanceCount;
    static int totalCount;
}
```

---

## static 메서드 주의사항

static 메서드는 객체 없이 호출될 수 있으므로 인스턴스 필드나 인스턴스 메서드를 직접 사용할 수 없다.

```java
class Sample {
    int value = 10;

    static void print() {
        // System.out.println(value); // 오류
    }
}
```

인스턴스 멤버를 사용하려면 객체를 생성해야 한다.

---

## final

`final`은 변경할 수 없음을 의미한다.

```java
final int MAX_COUNT = 100;
```

상수는 보통 `static final`로 선언한다.

```java
public static final int MAX_COUNT = 100;
```

---

## 내부 클래스

내부 클래스는 클래스 안에 선언된 클래스이다.

```java
class Outer {
    class Inner {
        void method() {
            System.out.println("inner");
        }
    }
}
```

관련성이 강한 클래스를 외부 클래스 내부에 묶어 표현할 때 사용한다.

---

## 핵심 정리

- `static` 멤버는 클래스에 소속되고 객체 없이 사용할 수 있다.
- static 메서드에서는 인스턴스 멤버를 직접 사용할 수 없다.
- `final` 변수는 한 번 값이 정해지면 변경할 수 없다.
- 내부 클래스는 특정 클래스와 강하게 관련된 클래스를 내부에 선언하는 방식이다.
