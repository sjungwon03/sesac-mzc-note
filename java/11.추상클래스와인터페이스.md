# 11. 추상 클래스와 인터페이스

## 추상 클래스

추상 클래스는 직접 객체를 만들 수 없고, 자식 클래스가 상속해서 완성해야 하는 클래스이다.

```java
abstract class Animal {
    abstract void sound();

    void eat() {
        System.out.println("먹는다");
    }
}
```

특징:

- `abstract` 키워드를 사용한다.
- 일반 필드와 일반 메서드를 가질 수 있다.
- 추상 메서드를 가질 수 있다.
- 직접 인스턴스화할 수 없다.

---

## 추상 메서드

추상 메서드는 선언만 있고 구현부가 없는 메서드이다.

```java
abstract void sound();
```

자식 클래스는 추상 메서드를 반드시 구현해야 한다.

```java
class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("멍멍");
    }
}
```

---

## 인터페이스

인터페이스는 클래스가 반드시 구현해야 할 기능의 규칙을 정의한다.

```java
interface Flyable {
    void fly();
}

class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("날다");
    }
}
```

특징:

- 클래스는 여러 인터페이스를 구현할 수 있다.
- 인터페이스의 추상 메서드는 기본적으로 `public abstract`이다.
- 인터페이스의 필드는 기본적으로 `public static final`이다.

---

## default 메서드와 static 메서드

인터페이스는 `default` 메서드와 `static` 메서드를 가질 수 있다.

```java
interface Printer {
    void print();

    default void ready() {
        System.out.println("ready");
    }

    static void info() {
        System.out.println("printer interface");
    }
}
```

---

## 추상 클래스와 인터페이스 비교

| 구분 | 추상 클래스 | 인터페이스 |
| --- | --- | --- |
| 목적 | 공통 속성과 기능 제공 | 구현 규칙 제공 |
| 키워드 | `extends` | `implements` |
| 다중 적용 | 불가능 | 가능 |
| 필드 | 인스턴스 필드 가능 | 상수 중심 |
| 생성자 | 가능 | 불가능 |

---

## 핵심 정리

- 추상 클래스는 공통 기능을 가진 미완성 부모 클래스이다.
- 추상 메서드는 자식 클래스가 반드시 구현해야 한다.
- 인터페이스는 기능 구현 규칙을 정의한다.
- 다중 구현이 필요하면 인터페이스를 사용한다.
