# AOP

## 1. AOP (Aspect Oriented Programming)

### 1.1 정의

- 관점 지향 프로그래밍
- 애플리케이션 전반에 반복되는 공통 부가 기능을 핵심 비즈니스 로직으로부터 분리하는 프로그래밍 기법
- 공통 기능을 한 곳에서 모듈화하여 관리

---

### 1.2 등장 배경

**반복되는 공통 코드:**
- 로그 출력
- 트랜잭션 처리
- 권한 체크
- 예외 처리
- 실행 시간 측정

**문제점:**
- 여러 비즈니스 메서드에 같은 코드가 반복됨
- 핵심 로직과 부가 기능이 섞여 코드가 복잡해짐
- 공통 기능 변경 시 여러 메서드를 모두 수정해야 함

**해결:**
- 반복되는 공통 코드를 잘라내어(Cross-cutting) 한 곳에서 관리
- 핵심 비즈니스 클래스는 자기 본연의 기능에만 집중

---

### 1.3 핵심 관심사와 공통 관심사

| 구분 | 설명 | 예시 |
|------|------|------|
| 핵심 관심사 | 실제 비즈니스 목적을 수행하는 코드 | 회원가입, 주문 생성, 결제 처리 |
| 공통 관심사 | 여러 기능에 반복적으로 필요한 부가 코드 | 로그, 트랜잭션, 권한, 예외 처리 |

---

### 1.4 AOP 적용 전/후

**AOP 적용 전:**
```java
public void order() {
  long start = System.currentTimeMillis();
  System.out.println("권한 체크");

  // 핵심 비즈니스 로직
  System.out.println("주문 처리");

  long end = System.currentTimeMillis();
  System.out.println("실행 시간: " + (end - start));
}
```

**AOP 적용 후:**
```java
public void order() {
  // 핵심 비즈니스 로직만 남김
  System.out.println("주문 처리");
}
```

## 2. AOP 핵심 개념

### 2.1 주요 용어

| 용어 | 의미 | 설명 |
|------|------|------|
| Aspect | 공통 부가 기능 모듈 | 분리해 낸 공통 기능을 담은 클래스 |
| Advice | 실제 부가 기능 코드 | 언제, 무엇을 실행할지 정의 |
| Join Point | Advice 적용 후보 지점 | 스프링에서는 메서드 호출 시점만 지원 |
| Pointcut | Advice 적용 대상 선정 규칙 | Join Point 중 실제 적용할 메서드 선택 |
| Weaving | Advice를 대상 메서드에 적용하는 과정 | 핵심 로직에 공통 기능을 끼워 넣는 과정 |
| Proxy | 대리 객체 | 실제 객체 대신 호출되어 Advice를 적용 |

---

### 2.2 Aspect

**정의:**
- 분리해 낸 공통 부가 기능 자체를 모듈화한 객체
- 별도 클래스로 구성
- `@Aspect` 어노테이션으로 Aspect임을 표시

**예시 공통 기능:**
- 실행 시간 측정
- 요청 로그 기록
- 트랜잭션 처리
- 권한 검사

---

### 2.3 Advice

**정의:**
- 실질적으로 수행해야 하는 부가 기능 코드
- 언제 공통 기능을 실행할지 결정

**적용 시점:**
- 메서드 실행 전
- 메서드 실행 후
- 메서드 실행 전후
- 정상 반환 후
- 예외 발생 후

---

### 2.4 Join Point

**정의:**
- Advice가 삽입될 수 있는 모든 후보 지점
- 스프링 AOP에서는 클래스의 메서드 호출 시점만 지원

**예시:**
- `MemberService.join()` 호출 시점
- `OrderService.order()` 호출 시점
- `PaymentService.pay()` 호출 시점

---

### 2.5 Pointcut

**정의:**
- Join Point 중 실제 Advice를 적용할 타겟 메서드를 선정하는 규칙

**예시:**
```java
execution(* com.example..*Service.*(..))
```

**의미:**
- `com.example` 패키지 아래
- 이름이 `Service`로 끝나는 클래스의
- 모든 메서드에 Advice 적용

---

## 3. Weaving과 Proxy

### 3.1 Weaving

**정의:**
- Advice를 비즈니스 메서드에 적용하는 과정
- 공통 기능을 핵심 로직 실행 흐름에 끼워 넣는 작업

---

### 3.2 Weaving 방식

| 방식 | 시점 | 설명 |
|------|------|------|
| 컴파일 타임 Weaving | 컴파일 시점 | 공통 코드가 삽입된 `.class` 파일 생성 |
| 클래스 로딩 타임 Weaving | 클래스 로딩 시점 | JVM이 클래스를 메모리에 올릴 때 AOP 적용 |
| 런타임 Weaving | 실행 시점 | 프록시 객체를 사용하여 AOP 적용 |

> 스프링 AOP는 런타임 Weaving 방식을 사용

---

### 3.3 Proxy 객체

**정의:**
- 실제 객체를 대신하는 대리 객체

**동작 방식:**
- 스프링 컨테이너에 실제 대상 객체(Target)를 그대로 등록하지 않음
- Advice가 부착된 Proxy 객체를 빈으로 등록
- Controller나 다른 Bean이 호출하는 객체는 실제 Service가 아니라 Proxy 객체
- Proxy가 공통 기능을 수행한 뒤 실제 객체 메서드를 호출

**흐름:**
```
Controller
    ↓
Proxy 객체
    ↓
Advice 실행
    ↓
실제 Service 메서드 실행
    ↓
Advice 실행
    ↓
결과 반환
```

## 4. Advice 종류

### 4.1 Advice 비교

| 종류 | 동작 시점 | 반환 타입 | 매개변수 타입 |
|------|-----------|-----------|---------------|
| `@Before` | 메서드 실행 이전 | `void` | `JoinPoint` |
| `@After` | 메서드 실행 이후 | `void` | `JoinPoint` |
| `@Around` | 메서드 실행 이전/이후 | `Object` | `ProceedingJoinPoint` |
| `@AfterReturning` | 예외 없이 정상 반환 후 | `void` | `JoinPoint` |
| `@AfterThrowing` | 예외 발생 후 | `void` | `JoinPoint` |

---

### 4.2 Advice 사용 기준

| 상황 | 권장 Advice |
|------|-------------|
| 메서드 실행 전에 로그를 남기고 싶을 때 | `@Before` |
| 성공/실패와 관계없이 실행 후 정리 작업이 필요할 때 | `@After` |
| 실행 전후 시간을 측정하거나 흐름을 제어할 때 | `@Around` |
| 정상 반환 값을 다루고 싶을 때 | `@AfterReturning` |
| 예외 발생 시 로깅/알림을 처리할 때 | `@AfterThrowing` |

---

## 5. `JoinPoint`

### 5.1 정의

- AOP가 적용되는 지점을 표현하는 객체
- Advice 메서드에서 현재 실행 중인 대상 메서드의 정보를 확인할 수 있음

### 5.2 특징

- 모든 Advice는 첫 번째 매개변수로 `JoinPoint` 사용 가능
- `@Around`는 반드시 `ProceedingJoinPoint` 사용
- `ProceedingJoinPoint`는 실제 대상 메서드를 실행하는 `proceed()` 메서드를 제공

---

### 5.3 주요 메서드

| 반환 타입 | 메서드 | 의미 |
|-----------|--------|------|
| `Object[]` | `getArgs()` | 대상 메서드로 전달된 argument 반환 |
| `Signature` | `getSignature()` | 대상 메서드의 시그니처 반환 |
| `Object` | `getTarget()` | 대상 객체 반환 |
| `String` | `toString()` | 대상 메서드 설명 문자열 반환 |

---

## 6. Pointcut 표현식

### 6.1 `execution()` 지시자

**정의:**
- 어떤 메서드에 Advice를 적용할지 지정하는 Pointcut 표현식

**기본 문법:**
```java
execution([접근제한자] [반환타입] 패키지.클래스.메소드(매개변수))
```

---

### 6.2 표현식 구성 요소

| 구분 | 예시 | 의미 |
|------|------|------|
| 접근 제한자 | `public`, `private` | 메서드 접근 제한자, 생략 가능 |
| 반환 타입 | `*`, `void`, `!void` | 메서드 반환 타입 |
| 패키지 | `com.example..` | 적용할 패키지 경로 |
| 클래스 | `*Service` | 적용할 클래스명 패턴 |
| 메서드 | `*`, `find*` | 적용할 메서드명 패턴 |
| 매개변수 | `*`, `..` | `*`는 1개, `..`는 개수 상관없음 |

---

### 6.3 Pointcut 예시

| 표현식 | 의미 |
|--------|------|
| `execution(* com.example..*Service.*(..))` | `com.example` 하위 모든 `Service` 클래스의 모든 메서드 |
| `execution(public * *(..))` | 모든 public 메서드 |
| `execution(* com.example.member.*.*(..))` | `com.example.member` 패키지의 모든 클래스/메서드 |
| `execution(* *..find*(..))` | 이름이 `find`로 시작하는 모든 메서드 |
| `execution(void com.example..*.*(..))` | 반환 타입이 `void`인 메서드 |

## 7. 스프링 AOP 사용하기

### 7.1 의존성 추가

**Gradle:**
```groovy
dependencies {
  implementation 'org.springframework.boot:spring-boot-starter-aop'
}
```

---

### 7.2 실행 시간 측정 Aspect 예시

```java
import org.aspectj.lang.ProceedingJoinPoint;
import org.aspectj.lang.annotation.Around;
import org.aspectj.lang.annotation.Aspect;
import org.springframework.stereotype.Component;

@Aspect
@Component
public class TimeTraceAspect {

  @Around("execution(* com.example..*Service.*(..))")
  public Object executeTimeTrace(ProceedingJoinPoint joinPoint) throws Throwable {
    long start = System.currentTimeMillis();

    System.out.println("AOP 실행 메서드: " + joinPoint.toString());

    try {
      Object result = joinPoint.proceed();
      return result;
    } finally {
      long finish = System.currentTimeMillis();
      long timeMs = finish - start;
      System.out.println("AOP 가로채기 종료(실행 시간: " + timeMs + "ms)");
    }
  }
}
```

---

### 7.3 코드 해설

| 코드 | 의미 |
|------|------|
| `@Aspect` | 이 클래스가 공통 부가 기능 모듈임을 표시 |
| `@Component` | 스프링 컨테이너에 Bean으로 등록 |
| `@Around(...)` | 대상 메서드 실행 전후를 감싸는 Advice |
| `ProceedingJoinPoint` | 실제 대상 메서드 실행을 제어하는 객체 |
| `joinPoint.proceed()` | 실제 대상 메서드 실행 |
| `finally` | 성공/예외와 관계없이 실행 시간 출력 |

---

## 8. AOP 적용 흐름

### 8.1 실행 흐름

```
1. 애플리케이션 실행
2. 스프링 컨테이너가 Aspect Bean 등록
3. Pointcut 조건에 맞는 대상 Bean 확인
4. 대상 Bean 대신 Proxy Bean 등록
5. Controller가 Service 호출
6. 실제 Service가 아니라 Proxy가 먼저 호출됨
7. Proxy가 Advice 실행
8. joinPoint.proceed()로 실제 Service 메서드 실행
9. 후처리 Advice 실행
10. 결과 반환
```

---

### 8.2 주의점

| 주의점 | 설명 |
|--------|------|
| Spring AOP 적용 지점 | 메서드 호출 시점만 지원 |
| 프록시 기반 | 실제 객체가 아니라 Proxy를 통해 호출될 때 AOP 적용 |
| 내부 메서드 호출 | 같은 클래스 내부에서 자기 메서드를 호출하면 AOP가 적용되지 않을 수 있음 |
| 과도한 사용 | 비즈니스 흐름을 숨길 수 있으므로 공통 관심사에만 사용 |
