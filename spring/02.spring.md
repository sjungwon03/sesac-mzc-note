# 프레임워크와 스프링

## 1. 프레임워크 (Framework)

### 1.1 정의

- 한글 번역: '뼈대'
- 애플리케이션의 기본 구조를 제공
- 개발 시 사용할 수 있는 미리 만들어진 클래스, 인터페이스 등을 제공
- 대표 프레임워크: 스트러츠, 스프링

### 1.2 특징

| 특징 | 설명 |
|------|------|
| 재사용성 | 이미 만들어진 기능 사용, 효율성 높고 품질 보장 |
| 개발 시간 | 처음부터 만들 필요 없어 시간 단축 |
| 표준화 | 개발 방법이 정해져 생산성 향상 |
| 유지보수 | 개발 후 유지보수 및 기능 확장 용이 |
| 진입 장벽 | 신입 개발자도 경력 개발자처럼 세련된 코드 작성 가능 |

## 2. 스프링 프레임워크 (Spring Framework)

### 2.1 정의

- 슬로건: "개발자에게 봄(Spring)을 선사하겠어요."
- 2004년 로드 존슨(Rod Johnson)이 만든 오픈소스 프레임워크
- 자바 기반 엔터프라이즈 애플리케이션 개발 담당
- 기존 **EJB(Enterprise JavaBeans)**를 대체
- 공식 홈페이지: https://spring.io

---

### 2.2 핵심 특징

| 특징 | 설명 |
|------|------|
| 경량 프레임워크 (Lightweight) | EJB에 비해 가벼움, 몇 개 모듈과 JAR로 구성 |
| 컨테이너 지원 (Spring Container) | 객체 생성/관리, Singleton, Prototype 등 다양한 객체 운용 |
| 제어의 역행 (IoC) | 객체 생성을 컨테이너가 담당, 낮은 결합도 유지 |
| 관점 지향 프로그래밍 (AOP) | 공통 모듈과 핵심 모듈 분리, 반복 로직 제거, 생산성 향상 |

---

### 2.3 스프링 프레임워크 모듈

| 모듈 | 설명 |
|------|------|
| spring-beans | 스프링 컨테이너를 이용한 객체 생성 기본 기능 |
| spring-context | 객체 생성, 라이프사이클 처리, 스키마 확장 |
| spring-aop | AOP 기능 제공 |
| spring-web | REST 클라이언트, 데이터 변환, 서블릿 필터, 파일 업로드 등 웹 개발 기반 기능 |
| spring-webmvc | MVC 프레임워크, 컨트롤러, 뷰 구현 |
| spring-websocket | 스프링 MVC에서 웹 소켓 연동 |
| spring-tx | 트랜잭션 처리 추상 레이어 |
| spring-jdbc | JDBC 프로그래밍 템플릿 |
| spring-orm | 하이버네이트, JPA, MyBatis 등 연동 지원 |
| spring-jms | JMS 서버 메시지 처리 템플릿, 어노테이션 |
| spring-context-support | 스케줄링, 메일 발송, 캐시 연동 등 부가 기능 |

## 3. 전자정부 표준프레임워크

### 3.1 정의

- 공공 기관 웹 애플리케이션에 공통적으로 사용하는 기반 기술과 라이브러리
- 공식 홈페이지: https://www.egovframe.go.kr/

### 3.2 특징

| 특징 | 설명 |
|------|------|
| 표준화 | 정부 관련 프로젝트 공통 개발 생태계 |
| 재사용성 | 개발된 컴포넌트/모듈 다른 프로젝트에서 재사용 가능 |
| 확장성 |新功能 추가/수정 용이 |
| 보안 | 공공 정보 시스템 필수 보안 기능 내장 |
| 기반 기술 | 자바 기반 Spring Framework 채택 |
