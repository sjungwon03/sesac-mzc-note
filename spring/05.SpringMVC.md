# Spring MVC

## 1. MVC 패턴

### 1.1 등장 배경

**초기 웹 개발 방식 (Model-1):**
- JSP 파일 하나에 요청 처리, 응답 처리, HTML 생성, DB 접근 코드가 모두 들어감
- 화면 코드와 비즈니스 로직이 섞여 스파게티 코드가 되기 쉬움
- 유지보수와 기능 확장이 어려움

**해결 방향:**
- 역할별 관심사를 분리하여 개발하는 MVC 패턴 등장

---

### 1.2 정의

- Model, View, Controller로 구성된 웹 애플리케이션 표준 디자인 패턴
- 데이터, 화면, 제어 흐름을 분리하여 독립적으로 개발하고 유지보수하는 구조
- Model-2 방식이라고도 부름

---

### 1.3 MVC 구성 요소

| 구성 요소 | 역할 | 예시 |
|-----------|------|------|
| Model | 데이터와 비즈니스 로직 처리 | DTO, Entity, Service, Repository |
| View | 클라이언트 화면 처리 | JSP, Thymeleaf, React, Vue |
| Controller | 요청을 받고 전체 흐름 제어 | Spring Controller |

---

### 1.4 Model

**정의:**
- 애플리케이션 데이터와 순수 비즈니스 로직을 담당하는 영역

**특징:**
- 화면이나 컨트롤러 정보를 알지 않음
- 순수 Java 중심으로 구성
- 데이터 객체와 비즈니스 로직의 집합

**예시:**
- DTO
- Entity
- Service
- Repository

---

### 1.5 View

**정의:**
- 클라이언트에게 보여줄 화면을 담당하는 영역

**특징:**
- Model이 넘겨준 데이터를 화면에 출력
- 서버 사이드 렌더링(SSR)에서는 JSP, Thymeleaf 같은 템플릿 엔진이 View 담당
- 클라이언트 사이드 렌더링(CSR)에서는 서버가 JSON 데이터를 보내고 React, Vue 등이 화면을 그림

| 렌더링 방식 | View 역할 |
|-------------|-----------|
| SSR | JSP, Thymeleaf 등 서버 템플릿 엔진 |
| CSR | React, Vue 등 프론트엔드 라이브러리 |

---

### 1.6 Controller

**정의:**
- 클라이언트 요청을 받아 전체 흐름을 제어하고 중재하는 영역

**역할:**
- 사용자 요청의 최초 진입점(Entry Point)
- 요청 파라미터 검증
- Service 호출을 통한 비즈니스 로직 처리
- 처리 결과를 어떤 View로 보여줄지 결정

---

### 1.7 MVC 흐름

```
클라이언트 요청
    ↓
Controller
    ↓
Service / Repository (Model)
    ↓
Controller
    ↓
View
    ↓
클라이언트 응답
```

---

## 2. Spring MVC

### 2.1 정의

- Spring Framework에서 제공하는 MVC 기반 웹 애플리케이션 개발 프레임워크
- Front Controller 패턴을 기반으로 동작
- 모든 요청을 `DispatcherServlet`이 먼저 받아 처리 흐름을 제어

---

### 2.2 `DispatcherServlet`

**정의:**
- Spring MVC의 핵심 서블릿
- 클라이언트의 모든 요청을 가장 먼저 중앙에서 받아내는 Front Controller

**역할:**
- 요청 URL을 분석
- 요청을 처리할 적절한 Controller(Handler)를 찾음
- Controller에 요청 처리 위임
- 처리 결과를 View 또는 Response Body로 응답

**특징:**
- 레거시 Spring MVC에서는 `/WEB-INF/web.xml`에 설정 등록
- 기본적으로 `servlet-context.xml` 설정 파일을 읽어 스프링 프레임워크 구동

---

### 2.3 Spring MVC 동작 흐름

```
클라이언트 요청
    ↓
DispatcherServlet
    ↓
HandlerMapping
    ↓
Controller
    ↓
Service
    ↓
Repository
    ↓
Controller
    ↓
ViewResolver
    ↓
View
    ↓
클라이언트 응답
```

---

### 2.4 주요 구성 요소

| 구성 요소 | 역할 |
|-----------|------|
| `DispatcherServlet` | 모든 요청을 처음 받는 Front Controller |
| `HandlerMapping` | 요청 URL에 맞는 Controller 탐색 |
| `Controller` | 요청 처리, Service 호출, View 결정 |
| `Service` | 비즈니스 로직 처리 |
| `Repository` | 데이터 접근 처리 |
| `ViewResolver` | View 이름을 실제 View 파일 경로로 변환 |
| `View` | 최종 화면 렌더링 |

## 3. 전통적 MVC 패턴 코드

### 3.1 DTO

**역할:**
- 화면과 계층 사이에서 데이터를 전달하는 객체

```java
import lombok.AllArgsConstructor;
import lombok.Getter;

@Getter
@AllArgsConstructor
public class StudentDto {
  private Long id;
  private String name;
  private String major;
}
```

---

### 3.2 Repository (DAO)

**역할:**
- 데이터베이스 접근 담당
- 실제 프로젝트에서는 SQL 또는 ORM을 통해 데이터 조회/저장 수행

```java
import org.springframework.stereotype.Repository;
import java.util.ArrayList;
import java.util.List;

@Repository
public class StudentRepository {

  public List<StudentDto> findAllStudents() {
    List<StudentDto> dbMock = new ArrayList<>();
    dbMock.add(new StudentDto(1L, "김철수", "컴퓨터공학과"));
    dbMock.add(new StudentDto(2L, "이영희", "정보통신학과"));
    return dbMock;
  }
}
```

---

### 3.3 Service

**역할:**
- 비즈니스 로직 처리
- Repository를 호출하여 필요한 데이터를 가져오거나 가공

```java
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;
import java.util.List;

@Service
@RequiredArgsConstructor
public class StudentService {

  private final StudentRepository studentRepository;

  public List<StudentDto> getStudentList() {
    return studentRepository.findAllStudents();
  }
}
```

---

### 3.4 Controller

**역할:**
- 클라이언트 요청을 받고 Service 호출
- 처리 결과를 Model에 담아 View로 전달

```java
import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import java.util.List;

@Controller
@RequiredArgsConstructor
public class StudentViewController {

  private final StudentService studentService;

  @GetMapping("/students-page")
  public String showStudentsPage(Model model) {
    List<StudentDto> studentList = studentService.getStudentList();
    model.addAttribute("students", studentList);
    return "students";
  }
}
```

---

### 3.5 View (HTML)

**역할:**
- Controller가 Model에 담은 데이터를 화면에 출력

```html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
  <head>
    <meta charset="UTF-8" />
    <title>학생 목록 (전통적 MVC)</title>
  </head>
  <body>
    <h1>학생 관리 시스템 목록</h1>
    <table border="1">
      <thead>
        <tr>
          <th>ID</th>
          <th>이름</th>
          <th>전공</th>
        </tr>
      </thead>
      <tbody>
        <tr th:each="student : ${students}">
          <td th:text="${student.id}">1</td>
          <td th:text="${student.name}">홍길동</td>
          <td th:text="${student.major}">컴퓨터공학</td>
        </tr>
      </tbody>
    </table>
  </body>
</html>
```

## 4. 계층별 책임 정리

| 계층 | 클래스 예시 | 책임 |
|------|-------------|------|
| View | `students.html` | 사용자 화면 출력 |
| Controller | `StudentViewController` | 요청 수신, Model 저장, View 반환 |
| Service | `StudentService` | 비즈니스 로직 처리 |
| Repository | `StudentRepository` | 데이터 접근 |
| DTO | `StudentDto` | 데이터 전달 |
