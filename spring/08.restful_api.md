# 8. RESTful API

## REST API란?

과거에는 서버가 HTML 화면까지 만들어서 브라우저에 전달했다(SSR). JSP, Thymeleaf 같은 방식이다. 그런데 스마트폰 앱과 React/Vue가 등장하면서 클라이언트가 직접 화면을 그리는 CSR 방식이 주류가 되었다. CSR 환경에서 서버는 화면이 아니라 **순수 데이터(JSON)**만 주면 된다. 이렇게 동작하는 서버를 REST API 서버라고 한다.

REST는 자원을 URI로 구분하고, 행위를 HTTP Method로 표현하는 통신 아키텍처다.

---

## REST API 설계 규칙

URI는 자원(명사)만 표현하고, 행위(동사)는 HTTP Method로 나타낸다.

- ❌ `/getMembers`, `/deleteMember/1`
- ⭕ `/members`, `/members/1`

---

## HTTP Method

| Method | 역할 | 멱등성 | Request Body |
| ------ | ---- | ------ | ------------ |
| GET | 조회 | O | X |
| POST | 생성 | X | O |
| PUT | 수정 | O | O |
| DELETE | 삭제 | O | X |

멱등성이 있다는 건 여러 번 요청해도 결과가 같다는 뜻이다. POST만 매번 새 데이터가 생성되므로 멱등하지 않다.

---

## 경로 변수 (`@PathVariable`)

URL 경로에 `{변수명}`을 넣고, `@PathVariable`로 값을 받는다.

```java
@GetMapping("/{bookId}")
public void getBookDetail(@PathVariable("bookId") Long bookId) {
    System.out.println("bookId: " + bookId);
}
```

경로 변수는 자원을 **식별**할 때 쓰고(`/books/15`), 쿼리 스트링은 **필터링/검색**할 때 쓴다(`/books?genre=it&page=2`). 경로 변수가 누락되면 404가 발생한다.

---

## `@RestController`

`@Controller` + `@ResponseBody` 합성 어노테이션이다. View를 반환하지 않고 데이터(JSON)만 응답하는 컨트롤러다.

---

## `ResponseEntity<T>`

`@RestController`만 쓰면 성공 시 항상 `200 OK`만 돌아온다. 실무에서는 상황에 따라 다른 상태 코드가 필요하다.

| 상황 | 상태 코드 |
| ---- | --------- |
| 정상 조회 | 200 OK |
| 새 리소스 생성 | 201 Created |
| 잘못된 요청 | 400 Bad Request |
| 권한 없음 | 403 Forbidden |
| 리소스 없음 | 404 Not Found |

`ResponseEntity`는 상태 코드, 헤더, 본문을 직접 제어할 수 있는 Wrapper 클래스다.

```java
@GetMapping("/{id}")
public ResponseEntity<UserDto> getUser(@PathVariable("id") Long id) {
    UserDto user = new UserDto(id, "홍길동");
    return ResponseEntity.ok(user);  // 200 OK
}

@PostMapping
public ResponseEntity<UserDto> createUser(@RequestBody UserDto userDto) {
    UserDto savedUser = new UserDto(99L, userDto.getName());
    return ResponseEntity.status(HttpStatus.CREATED).body(savedUser);  // 201 Created
}
```

## 실습: 회원 관리 REST API

```java
@Getter
@NoArgsConstructor
@AllArgsConstructor
public class MemberDto {
    private Long id;
    private String name;
    private String email;
}
```

```java
@RestController
@RequestMapping("/api/v1/members")
public class MemberController {

    private final List<MemberDto> memberStore = new ArrayList<>();

    public MemberController() {
        memberStore.add(new MemberDto(1L, "james dean", "a@test.com"));
        memberStore.add(new MemberDto(2L, "eric nam", "b@test.com"));
        memberStore.add(new MemberDto(3L, "jessica alba", "c@test.com"));
        memberStore.add(new MemberDto(4L, "kim jinsu", "d@test.com"));
        memberStore.add(new MemberDto(5L, "james cameron", "e@test.com"));
    }

    @GetMapping
    public ResponseEntity<List<MemberDto>> getAllMembers(
            @RequestParam(required = false) String name
    ) {
        if (name != null) {
            List<MemberDto> filtered = memberStore.stream()
                    .filter(m -> m.getName().contains(name))
                    .toList();
            return ResponseEntity.ok(filtered);
        }
        return ResponseEntity.ok(memberStore);
    }

    @GetMapping("/{id}")
    public ResponseEntity<MemberDto> getMemberById(@PathVariable Long id) {
        return memberStore.stream()
                .filter(m -> m.getId().equals(id))
                .findFirst()
                .map(m -> ResponseEntity.ok(m))
                .orElse(ResponseEntity.status(HttpStatus.NOT_FOUND).build());
    }

    @PostMapping
    public ResponseEntity<String> createMember(@RequestBody MemberDto memberDto) {
        memberStore.add(memberDto);
        return ResponseEntity.status(HttpStatus.CREATED).body("회원 등록 성공");
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<String> deleteMember(@PathVariable Long id) {
        boolean removed = memberStore.removeIf(m -> m.getId().equals(id));
        if (removed) {
            return ResponseEntity.ok("회원 삭제 완료");
        }
        return ResponseEntity.status(HttpStatus.NOT_FOUND).body("없는 회원");
    }
}
```

## 실습: 도서 관리 REST API

요구사항: `GET /api/v1/books` 전체 조회, `POST /api/v1/books` 도서 추가, `PUT /api/v1/books/{isbn}` 가격 수정

```java
@Getter
@NoArgsConstructor
@AllArgsConstructor
public class BookDto {
    private Long isbn;
    private String title;
    private int price;
}
```

```java
@RestController
@RequestMapping("/api/v1/books")
public class BookController {

    private final List<BookDto> bookStore = new ArrayList<>();

    public BookController() {
        bookStore.add(new BookDto(97889L, "이것이 자바다", 30000));
    }

    @GetMapping
    public ResponseEntity<List<BookDto>> getBooks() {
        return ResponseEntity.ok(bookStore);
    }

    @PostMapping
    public ResponseEntity<String> addBook(@RequestBody BookDto bookDto) {
        bookStore.add(bookDto);
        return ResponseEntity.status(HttpStatus.CREATED).body("도서 등록 완료");
    }

    @PutMapping("/{isbn}")
    public ResponseEntity<String> updateBookPrice(@PathVariable Long isbn, @RequestBody BookDto updateDto) {
        for (int i = 0; i < bookStore.size(); i++) {
            BookDto currentBook = bookStore.get(i);
            if (currentBook.getIsbn().equals(isbn)) {
                bookStore.set(i, new BookDto(isbn, currentBook.getTitle(), updateDto.getPrice()));
                return ResponseEntity.ok("도서 가격 수정 성공");
            }
        }
        return ResponseEntity.status(HttpStatus.NOT_FOUND).body("해당 도서를 찾을 수 없습니다.");
    }
}
```

## 핵심 정리

- REST API는 URI로 자원을 식별하고 HTTP Method로 행위를 표현한다.
- URI는 명사, 행위는 HTTP Method로 작성한다.
- `@RestController`는 데이터(JSON)만 응답하는 컨트롤러다.
- `ResponseEntity`로 상태 코드, 헤더, 본문을 직접 제어한다.
- 경로 변수(`@PathVariable`)는 자원 식별, 쿼리 스트링(`@RequestParam`)은 필터링에 사용한다.
