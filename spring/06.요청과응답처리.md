# 요청과 응답 처리

## 1. 요청 처리하기

### 1.1 요청 매핑 어노테이션

**정의:**
- 클라이언트 요청 URL과 컨트롤러 메서드를 연결하는 어노테이션

| 어노테이션 | 설명 | 예시 |
|-----------|------|------|
| `@RequestMapping` | 가장 기본적인 요청 매핑 어노테이션, URL과 HTTP 메서드 함께 명시 | `@RequestMapping(value = "/api/test", method = RequestMethod.GET)` |
| `@GetMapping` | GET 요청 전용 축약형 어노테이션 | `@GetMapping("/api/test")` |
| `@PostMapping` | POST 요청 전용 축약형 어노테이션 | `@PostMapping("/api/test")` |

---

### 1.2 요청 파라미터 바인딩

**정의:**
- 클라이언트가 보낸 데이터를 컨트롤러 메서드 매개변수로 자동 전달하는 기능

**주요 방식:**

| 방식 | 사용 상황 | 특징 |
|------|-----------|------|
| `HttpServletRequest` | 레거시/저수준 처리 | 직접 꺼내고 직접 형변환 |
| `@RequestParam` | 단일 파라미터 처리 | 쿼리 스트링/폼 데이터 1:1 매핑 |
| 커맨드 객체 | 파라미터가 많을 때 | DTO로 한 번에 바인딩 |
| `@RequestBody` | JSON 요청 처리 | HTTP Body의 JSON을 객체로 변환 |

---

## 2. `HttpServletRequest` 직접 사용

### 2.1 특징

- 자바 웹 개발의 핵심 클래스인 `HttpServletRequest`를 직접 사용
- 모든 값이 문자열로 반환됨
- 숫자 데이터가 필요하면 직접 형 변환 필요

### 2.2 예시 HTML

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>HTTP 요청 테스트 화면</title>
  </head>
  <body>
    <a href="/api/v1/legacy-get?title=어린왕자&page=50">GET 요청</a>
  </body>
</html>
```

### 2.3 예시 Controller

```java
@Controller
public class RequestController {

  @RequestMapping(value = "/api/v1/legacy-get", method = RequestMethod.GET)
  public void legacyGet(HttpServletRequest request) {
    String title = request.getParameter("title");
    String pageStr = request.getParameter("page");
    int page = (pageStr != null) ? Integer.parseInt(pageStr) : 0;

    System.out.println("title: " + title + ", page: " + page);
  }
}
```

## 3. `@RequestParam`

### 3.1 특징

- 쿼리 스트링 또는 `<form>` 태그로 전송되는 데이터를 1:1로 매핑
- `int`, `Long` 등 기본 타입으로 자동 형 변환 지원
- 파라미터명과 변수명이 다르면 `name` 속성 사용
- 필수 여부와 기본값 설정 가능

### 3.2 주요 속성

| 속성 | 설명 | 예시 |
|------|------|------|
| `name` | 요청 파라미터 이름 지정 | `@RequestParam(name = "title")` |
| `required` | 필수 여부 지정 | `required = false` |
| `defaultValue` | 값이 없을 때 기본값 지정 | `defaultValue = "1"` |

### 3.3 예시 HTML

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>HTTP 요청 테스트 화면</title>
  </head>
  <body>
    <a href="/api/v1/request-param?title=소나기&page=100">GET 요청</a>

    <form action="/api/v1/request-param">
      제목: <input type="text" name="title" value="소나기" /><br />
      페이지: <input type="number" name="page" value="100" /><br />
      <button type="submit">GET 요청</button>
    </form>
  </body>
</html>
```

### 3.4 예시 Controller

```java
@Controller
public class RequestController {

  @GetMapping("/api/v1/request-param")
  public void springRequestParam(
      @RequestParam(name = "title") String boardTitle,
      @RequestParam(required = false, defaultValue = "1") int page
  ) {
    System.out.println("title: " + boardTitle + ", page: " + page);
  }
}
```

## 4. 커맨드 객체 (Command Object)

### 4.1 정의

- 쿼리 스트링이나 폼 파라미터를 하나의 자바 객체로 한 번에 받는 방식
- DTO 바인딩 방식이라고도 볼 수 있음

### 4.2 특징

- 별도 어노테이션 없이 스프링이 Setter를 호출하여 값 주입
- 요청 파라미터 이름과 객체 필드명이 같아야 자동 매칭
- 커맨드 객체는 `Model`에 자동 저장됨

---

### 4.3 예시 HTML

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>HTTP 요청 테스트 화면</title>
  </head>
  <body>
    <a href="/api/v1/command-object?title=홍길동전&page=200&author=허균">
      GET 요청
    </a>

    <form action="/api/v1/command-object">
      제목: <input type="text" name="title" value="홍길동전" /><br />
      페이지: <input type="number" name="page" value="200" /><br />
      저자: <input type="text" name="author" value="허균" />
      <button type="submit">GET 요청</button>
    </form>
  </body>
</html>
```

### 4.4 예시 DTO

```java
import lombok.Getter;
import lombok.Setter;
import lombok.ToString;

@Getter
@Setter
@ToString
public class BookRequest {
  private String title;
  private int page;
  private String author;
}
```

### 4.5 예시 Controller

```java
@Controller
public class RequestController {

  @GetMapping("/api/v1/command-object")
  public void springCommandObject(BookRequest bookRequest) {
    System.out.println("수신 Command Object: " + bookRequest.toString());
  }
}
```

## 5. `@RequestBody`

### 5.1 정의

- HTTP Request Body에 담긴 JSON 데이터를 자바 객체로 변환하는 방식
- REST API 서버 구축 시 표준적으로 사용

### 5.2 특징

- 주로 POST 요청에서 사용
- JSON 바인딩 방식
- 역직렬화(Deserialization): JSON 문자열 → 자바 객체
- 내부적으로 `HttpMessageConverter`와 Jackson 라이브러리 사용

---

### 5.3 예시 HTML

```html
<!DOCTYPE html>
<html lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>HTTP 요청 테스트 화면</title>
  </head>
  <body>
    <button onclick="sendJson()">POST JSON 요청</button>
    <script>
      function sendJson() {
        const book = {
          title: "태백산맥",
          page: 99,
        };

        fetch("/api/v1/json-post", {
          method: "POST",
          headers: {
            "Content-Type": "application/json",
          },
          body: JSON.stringify(book),
        });
      }
    </script>
  </body>
</html>
```

### 5.4 예시 Controller

```java
@Controller
public class RequestController {

  @PostMapping("/api/v1/json-post")
  public void springJsonPost(@RequestBody BookRequest bookRequest) {
    System.out.println("JSON 수신 객체: " + bookRequest.toString());
  }
}
```

## 6. 응답 처리하기

### 6.1 응답 방식

| 응답 방식 | 설명 | 대표 기술 |
|-----------|------|-----------|
| 화면 반환 | 클라이언트에게 보여줄 HTML 화면 반환 | SSR, Forward, Redirect |
| 데이터 반환 | HTML 전환 없이 데이터만 Response Body에 반환 | CSR, REST API, JSON |

## 7. 화면(View) 반환 방식

### 7.1 `ModelAndView`

**정의:**
- 뷰 이름과 데이터를 하나의 객체에 담아 반환하는 Spring MVC Legacy 방식

**특징:**
- `addObject()`로 데이터 저장
- `setViewName()`으로 View 이름 지정
- View 이름은 `ViewResolver`가 Prefix/Suffix를 붙여 실제 HTML 파일을 찾음

**예시 흐름:**
```
반환 뷰 이름: result-view
Prefix: src/main/resources/templates/
Suffix: .html
최종 파일: src/main/resources/templates/result-view.html
```

**예시 Controller:**
```java
@GetMapping("/page-mav")
public ModelAndView responseModelAndView() {
  ModelAndView mav = new ModelAndView();
  mav.addObject("name", "홍길동");
  mav.addObject("age", 30);
  mav.setViewName("result-view");
  return mav;
}
```

---

### 7.2 `Model`과 `String` 반환

**정의:**
- 현재 가장 표준적으로 사용하는 View 반환 방식

**특징:**
- `Model`은 HTML로 보낼 데이터를 담는 Map 구조 객체
- `model.addAttribute("key", value)`로 데이터 저장
- 반환 문자열은 View 이름으로 처리
- `ViewResolver`가 실제 HTML 파일 경로로 변환

**예시 Controller:**
```java
@GetMapping("/page-string")
public String responseString(Model model) {
  model.addAttribute("name", "홍길동");
  model.addAttribute("age", 30);
  return "result-view";
}
```

---

### 7.3 `void` 반환

**정의:**
- 컨트롤러 메서드의 반환 값이 없을 때 요청 주소를 View 이름으로 간주하는 방식

**예시:**
- 요청 URL: `/member/list`
- 찾는 View: `src/main/resources/templates/member/list.html`

**예시 Controller:**
```java
@GetMapping("/result-view")
public void responseVoid(Model model) {
  model.addAttribute("name", "홍길동");
  model.addAttribute("age", 30);
}
```

---

### 7.4 Thymeleaf View 예시

```html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org" lang="ko">
  <head>
    <meta charset="UTF-8" />
    <title>Model 데이터 수신 결과</title>
  </head>
  <body>
    <ul>
      <li>
        <strong>이름:</strong>
        <span th:text="${name}">여기에 이름이 출력됩니다.</span>
      </li>
      <li>
        <strong>나이:</strong>
        <span th:text="${age}">여기에 나이가 출력됩니다.</span>
        <span>세</span>
      </li>
    </ul>
  </body>
</html>
```

## 8. Redirect 처리

### 8.1 Redirect 문법

**정의:**
- `redirect:`으로 시작하는 문자열을 반환하면 스프링은 Redirect로 처리
- `redirect:` 뒤에는 View 이름이 아니라 이동할 URL을 작성

**예시:**
```java
@PostMapping("/board/write")
public String writePost() {
  // DB에 게시글 저장 등 비즈니스 로직 수행
  return "redirect:/board/list";
}

@GetMapping("/board/list")
public String listPost() {
  return "list";
}
```

---

### 8.2 Redirect 시 데이터 유실 문제

**문제:**
- Redirect는 요청을 두 번 보내는 방식
- 첫 번째 요청의 `Model` 데이터는 두 번째 요청에서 사라짐
- 저장 성공 메시지 같은 데이터를 전달하려면 별도 방법 필요

---

### 8.3 `RedirectAttributes`

**정의:**
- Redirect 목적지까지 데이터를 전달하기 위한 객체
- 컨트롤러 메서드 매개변수로 자동 주입 가능

**전달 방식:**

| 메서드 | 방식 | 특징 | 용도 |
|--------|------|------|------|
| `addAttribute()` | Query String | URL 뒤에 파라미터로 붙음, 새로고침해도 유지 | 검색 조건, 페이지 번호 |
| `addFlashAttribute()` | Session 임시 저장 | 목적지 도착 후 한 번 사용하고 소멸 | 저장 성공 메시지 |

### 8.4 예시 Controller

```java
@Controller
public class RedirectSampleController {

  @PostMapping("/board/save")
  public String saveProcess(
      @RequestParam("content") String content,
      RedirectAttributes redirectAttributes
  ) {
    // DB 저장 로직 수행
    redirectAttributes.addAttribute("page", 1);
    redirectAttributes.addFlashAttribute("msg", "게시글 등록 성공");
    return "redirect:/board/list";
  }

  @GetMapping("/board/list")
  public String listPage(
      @RequestParam(value = "page", required = false) int page,
      Model model
  ) {
    System.out.println("page: " + page);
    System.out.println("msg: " + model.getAttribute("msg"));
    return "board-list";
  }
}
```

## 9. 데이터(Body) 반환 방식

### 9.1 정의

- 화면으로 이동하지 않고 자바 데이터를 HTTP Response Body에 담아 반환하는 방식
- 요청 페이지는 그대로 유지되고 데이터만 응답
- 비동기 요청 또는 REST API에서 주로 사용

### 9.2 `@ResponseBody`

**특징:**
- 반환 값을 `ViewResolver`로 처리하지 않음
- `HttpMessageConverter`가 응답 처리
- 자바 객체(DTO)를 반환하면 Jackson이 JSON 문자열로 자동 변환

### 9.3 예시 DTO

```java
@NoArgsConstructor
@AllArgsConstructor
@Setter
@Getter
public class MemberDto {
  private String name;
  private String major;
}
```

### 9.4 예시 Controller

```java
@GetMapping("/api/v1/data")
@ResponseBody
public MemberDto responseBodyData() {
  return new MemberDto("홍길동", "computer");
}
```

**응답 JSON:**
```json
{"name":"홍길동", "major":"computer"}
```

## 10. 요청/응답 방식 비교

### 10.1 요청 데이터 처리 비교

| 방식 | 입력 데이터 | 장점 | 주의점 |
|------|-------------|------|--------|
| `HttpServletRequest` | Query/Form | 서블릿 원리 이해에 좋음 | 직접 형변환 필요 |
| `@RequestParam` | Query/Form | 단일 값 처리 간단 | 파라미터 많으면 복잡 |
| 커맨드 객체 | Query/Form | 여러 값 DTO로 묶기 좋음 | Setter 필요 |
| `@RequestBody` | JSON Body | REST API 표준 | `Content-Type: application/json` 필요 |

### 10.2 응답 처리 비교

| 방식 | 반환 대상 | 특징 | 사용 상황 |
|------|-----------|------|-----------|
| `ModelAndView` | View | 레거시 방식, 데이터와 View를 객체로 묶음 | 구형 Spring MVC |
| `Model` + `String` | View | 현재 표준 방식 | SSR 화면 반환 |
| `void` | View | 요청 URL을 View 이름으로 사용 | 단순 화면 연결 |
| `redirect:` | URL | 클라이언트 재요청 | DB 변경 후 목록 이동 |
| `@ResponseBody` | Data | JSON 등 Body 응답 | REST API, 비동기 통신 |

## 핵심 요약

| 개념 | 핵심 내용 |
|------|-----------|
| 요청 매핑 | URL과 Controller 메서드 연결 |
| `@RequestMapping` | URL과 HTTP 메서드를 함께 명시 |
| `@GetMapping` / `@PostMapping` | GET/POST 전용 축약형 |
| `HttpServletRequest` | 저수준 요청 처리, 직접 형변환 필요 |
| `@RequestParam` | 단일 파라미터 1:1 바인딩 |
| 커맨드 객체 | 여러 파라미터를 DTO로 한 번에 바인딩 |
| `@RequestBody` | JSON Body를 자바 객체로 변환 |
| `ModelAndView` | View 이름과 데이터를 하나의 객체로 반환 |
| `Model` + `String` | 현재 표준 View 반환 방식 |
| Redirect | `redirect:/url` 문법 사용 |
| `RedirectAttributes` | Redirect 목적지까지 데이터 전달 |
| `@ResponseBody` | View 없이 Response Body에 데이터 반환 |
