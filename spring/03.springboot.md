# 스프링 부트 (Spring Boot)

## 1. 스프링 부트 개요

### 1.1 등장 배경

| 시기 | 내용 |
|------|------|
| 2004.03 | 스프링 프레임워크 1.0 공식 출시 (EJB 복잡성 대폭 감소) |
| 이후 | 여전히 개발자가 직접 설정해야 하는 불편함 존재 |

**문제점:**
- XML 설정 지옥 (Configuration Hell)
- Dependency 충돌

**해결:**
- 스프링 부트는 개발 편의성과 생산성 향상을 위한 현대적인 프레임워크
- 환경 변수, 빈(Bean) 등록 등 작업 자동화
- 참고: 스프링 부트와 구분하기 위해 기존 스프링 프레임워크를 **스프링 레거시**라 표현

---

### 1.2 스프링 레거시 vs 스프링 부트 비교

| 항목 | 스프링 레거시 (Spring Framework) | 스프링 부트 (Spring Boot) |
|------|----------------------------------|---------------------------|
| 설정 | XML 또는 Java 설정 직접 작성 | Auto-Configuration 자동 설정 |
| WAS | 외부 Tomcat 설치·설정 필요 | 내장 WAS 포함 (Embedded Tomcat) |
| 의존성 | 버전 충돌 직접 해결 | `spring-boot-starter`로 자동 관리 (Parent POM) |
| 빈 등록 | 컴포넌트 스캔 범위 직접 명시 | `@SpringBootApplication`으로 자동 스캔 |
| 배포 | WAR 파일 패키징 후 외부 WAS 배포 | 단일 JAR 파일로 `java -jar` 실행 |

---

### 1.3 예시 비교: UTF-8 인코딩 필터

**스프링 레거시 (`web.xml`):**
```xml
<filter>
  <filter-name>encodingFilter</filter-name>
  <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
  <init-param>
    <param-name>encoding</param-name>
    <param-value>UTF-8</param-value>
  </init-param>
  <init-param>
    <param-name>forceEncoding</param-name>
    <param-value>true</param-value>
  </init-param>
</filter>
<filter-mapping>
  <filter-name>encodingFilter</filter-name>
  <url-pattern>/*</url-pattern>
</filter-mapping>
```

**스프링 부트 (`application.yml`):**
```yaml
server:
  servlet:
    encoding:
      charset: UTF-8
      enabled: true
      force: true
```

> 참고: 스프링 부트는 기본적으로 UTF-8 인코딩 필터가 활성화되어 있어 설정 자체가 불필요

## 2. `@SpringBootApplication`

### 2.1 정의

- 스프링 부트 프로젝트 시작점
- 자동 설정, 컴포넌트 스캔 시작
- 루트 패키지에 하나의 메인 클래스에 부착

### 2.2 3가지 합성 Annotation

| 어노테이션 | 역할 |
|-----------|------|
| `@SpringBootConfiguration` | 스프링 부트 설정 빈 명시 |
| `@EnableAutoConfiguration` | 자동 설정 메커니즘 활성화 |
| `@ComponentScan` | 현재 패키지 및 하위 패키지 `@Component` 계열 스캔 |

### 2.3 예시

```java
package com.example.myapp;

@SpringBootApplication
public class LectureApplication {
  public static void main(String[] args) {
    // 내장 톰캣 구동 + 스프링 컨테이너 초기화
    SpringApplication.run(LectureApplication.class, args);
  }
}
```

## 4. Spring Boot 프로젝트 생성 및 실행

### 4.1 프로젝트 생성 단계

| 단계 | 설정 |
|------|------|
| 1 | 명령 팔레트 (`Ctrl+Shift+P` 또는 `F1`) |
| 2 | `Spring Initializr: Create a Gradle Project…` |
| 3 | Spring Boot Version: `3.5.x` (최신 안정) |
| 4 | Language: `Java` |
| 5 | Group Id: `com.example` |
| 6 | Artifact Id: `demo` |
| 7 | Package Name: `com.example.demo` |
| 8 | Packaging Type: `Jar` |
| 9 | Java Version: `21` |
| 10 | Dependencies: `Spring Web`, `Lombok` |

---

| 파일/폴더 | 분류 | 설명 |

| `DemoApplication.java` | 실행 파일 | Entry Point, `main` 메서드, `@SpringBootApplication` |
| `src/main/resources` | 설정 및 자원 | 설정 파일, 정적 자원 |

| `src/test` | 테스트 코드 | JUnit 테스트 클래스 |
| `.gitattributes` | Git 설정 | 줄바꿈 문자(LF/CRLF) 충돌 방지 |

| `gradlew.bat` | 실행 스크립트 | Windows용 Gradle 실행 |
| `HELP.md` | 가이드 문서 | 참고용, 삭제 무방 |

### 4.4 `spring-boot-starter-web` 핵심 디펜던시

| 디펜던시 | 역할 |
|----------|------|
| `spring-boot-starter` | 최상위 부모, 자동 설정, 코어 빈 리포지토리 |

## 5. 빌드 도구 (Build Tool)

### 5.1 등장 배경

- 초기 자바 개발: 컴파일, 라이브러리 다운로드/배치 모두 개발자가 직접 수행
- 문제점: 라이브러리 간 버전 충돌 빈번, 조합 테스트에 시간 소모
- 해결: 빌드 도구가 라이브러리 버전 자동 관리 및 빌드 자동화

### 5.2 핵심 역할

| 역할 | 설명 |
|------|------|
| 의존성 관리 | 라이브러리 이름/버전 명시 시 원격 저장소에서 자동 다운로드 |
| 빌드 자동화 | 컴파일, 테스트, 배포 파일(`.jar`/`.war`) 생성 자동화 |

### 5.3 주요 빌드 도구

- Maven (메이븐)
- Gradle (그래들)

---

### 5.4 Gradle

**정의:**
- 안드로이드 공식 빌드 도구
- 스프링 부트 생태계 표준 빌드 자동화 시스템

**특징:**

| 특징 | 설명 |
|------|------|
| 코드 기반 설정 | Groovy/Kotlin 사용, `if`/`for` 제어문 가능 |
| 점진적 빌드 | 변경된 부분만 컴파일 |
| 빌드 캐시 | 실행된 태스크 결과 재사용 |

---

### 5.5 Gradle vs Maven

| 항목 | Maven | Gradle |
|------|-------|--------|
| 설정 파일 | XML (`pom.xml`) | Groovy/Kotlin DSL (`build.gradle`) |
| 가독성 | 길고 복잡함 (Verbose) | 간결하고 직관적 |
| 동적 제어 | 불가능 (정적 선언) | 가능 (조건문, 변수) |
| 빌드 속도 | 느림 | 빠름 (캐싱, 점진적 빌드, 2~10배) |
| 커스텀 태스크 | 어려움 | 쉬움 |

**코드 비교:**

Maven:
```xml
<dependency>
    <groupId>tools.jackson.core</groupId>
    <artifactId>jackson-databind</artifactId>
    <version>3.1.3</version>
    <scope>compile</scope>
</dependency>
```

Gradle:
```groovy
implementation("tools.jackson.core:jackson-databind:3.1.3")
```

## 6. 스프링 부트 Gradle 핵심 파일

### 6.1 `build.gradle`

**역할:**
- 플러그인, 의존성, 빌드 태스크 정의

**예시:**
```groovy
plugins {
    id 'java'
    id 'org.springframework.boot' version '3.5.0'
    id 'io.spring.dependency-management' version '1.1.4'
}

group = 'com.education'
version = '0.0.1-SNAPSHOT'
java { sourceCompatibility = '17' }

repositories {
    mavenCentral()  // 원격 저장소
}

dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-web'
    compileOnly 'org.projectlombok:lombok'
    annotationProcessor 'org.projectlombok:lombok'
    testImplementation 'org.springframework.boot:spring-boot-starter-test'
}
```

---

### 6.2 `gradlew` / `gradlew.bat` (Gradle Wrapper)

**역할:**
- PC에 Gradle 미설치 시 프로젝트에 Gradle 다운로드 후 빌드

**빌드 명령:**

| 명령 | 설명 |
|------|------|
| `./gradlew build` | 프로젝트 빌드 |
| `./gradlew bootRun` | 빌드 + 서버 실행 |

**참고: Permission denied (Linux/Mac)**
```bash
chmod +x gradlew
```
