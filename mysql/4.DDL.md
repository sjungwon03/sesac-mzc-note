# DDL

## 1. DDL이란?

**Data Definition Language (데이터 정의어)**의 약자로, 데이터베이스 구조를 정의하는 SQL 문이다.

**특징:**
- 테이블, 뷰 등 데이터베이스 객체를 생성/수정/삭제
- 데이터 간의 관계 정의
- **실행 후 작업 취소 불가능**

**주요 명령어:**
- `CREATE`: 객체 생성
- `ALTER`: 객체 수정
- `DROP`: 객체 삭제
- `TRUNCATE`: 데이터 및 저장 공간 삭제

---

## 2. 데이터베이스 구축 절차

```
요구사항 분석 → 개념적 설계 → 논리적 설계 → 물리적 설계 → 구현
```

## 3. 테이블 및 칼럼 생성

### 3.1 테이블 생성 개념

테이블 구조를 정의하고 데이터를 저장하기 위한 공간을 할당하는 과정

### 3.2 테이블 이름 규칙

- 소문자 사용 권장
- 숫자 사용 지양
- snake_case 방식 사용 (`_`로 단어 연결)
- 테이블임을 알리는 prefix/suffix 권장 (예: `tbl_user`)

### 3.3 칼럼 이름 규칙

- 소문자 사용 권장
- 숫자 사용 지양
- snake_case 방식 사용
- PK는 `id` 등 식별 키워드 포함 권장
- 지나친 줄임말 지양 (예: `department_id` → `dpt_id` ❌)

### 3.4 무결성 제약 조건

**무결성:** 데이터베이스의 데이터에 문제가 없는 상태

**제약 조건 종류:**

| 제약조건 | 설명 |
|---------|------|
| `NOT NULL` | NULL 값 허용하지 않음 (입력 필수) |
| `UNIQUE` | 중복 값 허용하지 않음 |
| `CHECK` | 유효성 규칙 준수 |
| `PRIMARY KEY` | 기본키 지정 (NULL 불가, 중복 불가) |
| `FOREIGN KEY` | 외래키 지정 (다른 테이블 참조) |

### 3.5 AUTO_INCREMENT

**특징:**
- 1부터 자동으로 1씩 증가하는 정수 값
- 반드시 PRIMARY KEY로 지정해야 함

**조작 방법:**

**1) 시작 값 변경**
```sql
ALTER TABLE 테이블명 AUTO_INCREMENT = 1000;
```

**2) 증가 값 변경**
```sql
SET @@auto_increment_increment = 2;
```

### 3.6 테이블 생성 문법

```sql
CREATE TABLE 테이블_이름 (
    칼럼명1 데이터_타입 제약조건,
    칼럼명2 데이터_타입 제약조건,
    ...
);
```

### 3.7 테이블 구조 확인

```sql
DESC 테이블_이름;
-- 또는
DESCRIBE 테이블_이름;
```

## 4. 테이블 및 칼럼 변경 (ALTER)

### 4.1 칼럼 추가

**단일 칼럼 추가**
```sql
ALTER TABLE 테이블_이름 
ADD COLUMN 칼럼_이름 데이터_타입 [제약조건] [DEFAULT 값];
```

**다중 칼럼 추가**
```sql
ALTER TABLE 테이블_이름
    ADD COLUMN 칼럼1 데이터_타입 [제약조건],
    ADD COLUMN 칼럼2 데이터_타입 [제약조건],
    ADD COLUMN 칼럼3 데이터_타입 [제약조건];
```

### 4.2 칼럼 수정

```sql
ALTER TABLE 테이블_이름 
MODIFY COLUMN 칼럼_이름 데이터_타입 [제약조건];
```

### 4.3 칼럼 삭제

```sql
ALTER TABLE 테이블_이름 
DROP COLUMN 칼럼_이름;
```

### 4.4 칼럼 이름 변경

```sql
ALTER TABLE 테이블_이름 
RENAME COLUMN 기존_칼럼_이름 TO 새_칼럼_이름;
```

### 4.5 테이블 이름 변경

**방법 1:**
```sql
RENAME TABLE 기존_테이블_이름 TO 새_테이블_이름;
```

**방법 2:**
```sql
ALTER TABLE 기존_테이블_이름 RENAME 새_테이블_이름;
```

## 5. 테이블 삭제 (DROP, TRUNCATE)

### 5.1 TRUNCATE

데이터만 삭제, 구조 유지 (복구 불가능)

```sql
TRUNCATE TABLE 테이블_이름;
```

### 5.2 DROP

테이블 완전 삭제 (복구 불가능)

```sql
DROP TABLE 테이블_이름;
```

### 5.3 존재 여부 확인 후 삭제

```sql
DROP TABLE IF EXISTS 테이블_이름;
```

### 5.4 TRUNCATE vs DROP

| 명령어 | 데이터 | 구조 | 복구 |
|--------|--------|------|------|
| `TRUNCATE` | 삭제 | 유지 | 불가 |
| `DROP` | 삭제 | 삭제 | 불가 |

## 6. 데이터 타입

### 6.1 정수형

| 타입 | 크기 | 범위 |
|------|------|------|
| `TINYINT` | 1바이트 | -128 ~ 127 |
| `SMALLINT` | 2바이트 | -32,758 ~ 32,767 |
| `INT` | 4바이트 | -2,147,483,648 ~ 2,147,483,647 |
| `BIGINT` | 8바이트 | 약 -900경 ~ 900경 |

### 6.2 실수형

| 타입 | 크기 | 표현 범위 |
|------|------|----------|
| `FLOAT` | 4바이트 | 소수점 아래 7자리 |
| `DOUBLE` | 8바이트 | 소수점 아래 15자리 |

### 6.3 문자형

| 타입 | 특징 | 최대 크기 |
|------|------|----------|
| `CHAR(size)` | 고정 길이 | 255자 |
| `VARCHAR(size)` | 가변 길이 | 16,383자 |

**CHAR vs VARCHAR:**

| 구분 | CHAR | VARCHAR |
|------|------|---------|
| 저장 방식 | 고정 길이 | 가변 길이 |
| 공간 효율 | 지정된 크기만큼 항상 사용 | 실제 데이터 크기만 사용 |
| 성능 | 검색 빠름 | 저장 효율적 |
| 사용 예 | 고정 길이 데이터 (전화번호, 주민번호 등) | 가변 길이 데이터 (이름, 이메일 등) |

### 6.4 날짜형

| 타입 | 크기 | 형식 |
|------|------|------|
| `DATE` | 3바이트 | YYYY-MM-DD |
| `TIME` | 3바이트 | HH:MM:SS |
| `DATETIME` | 8바이트 | YYYY-MM-DD HH:MM:SS |

### 6.5 대용량 데이터

| 타입 | 용도 | 최대 크기 |
|------|------|----------|
| `TEXT` | 텍스트 | 65,536자 |
| `LONGTEXT` | 텍스트 | 4,294,967,295자 |
| `BLOB` | 바이너리 | 65,536바이트 |
| `LONGBLOB` | 바이너리 | 4,294,967,295바이트 |

## 7. 실습 예제

### 7.1 데이터베이스 생성

```sql
CREATE DATABASE IF NOT EXISTS db_ddl;
```

### 7.2 테이블 생성 예제

**요구사항:**
- `cust_id`: 기본키, 자동 증가
- `cust_name`: 필수 입력
- `phone`: 중복 불가
- `age`: 0~100 사이만 가능
- `join_dt`: 기본값 현재 날짜

**SQL:**
```sql
CREATE TABLE customers (
    cust_id   INT         PRIMARY KEY AUTO_INCREMENT,
    cust_name VARCHAR(30) NOT NULL,
    phone     VARCHAR(30) UNIQUE,
    age       SMALLINT    CHECK (age BETWEEN 0 AND 100),
    join_dt   DATE        DEFAULT (CURRENT_DATE)
);
```

## 8. 핵심 요약

### DDL 명령어
1. **CREATE:** 데이터베이스 객체 생성
2. **ALTER:** 객체 구조 수정
3. **DROP:** 객체 완전 삭제
4. **TRUNCATE:** 데이터만 삭제

### 무결성 제약 조건
- **NOT NULL:** NULL 불가
- **UNIQUE:** 중복 불가
- **CHECK:** 조건 만족
- **PRIMARY KEY:** 기본키
- **FOREIGN KEY:** 외래키

### 데이터 타입 선택 가이드
- **정수:** `TINYINT` < `SMALLINT` < `INT` < `BIGINT`
- **실수:** `FLOAT` (7자리) < `DOUBLE` (15자리)
- **문자:** 고정 길이는 `CHAR`, 가변 길이는 `VARCHAR`
- **날짜:** 날짜만 `DATE`, 시간만 `TIME`, 둘 다 `DATETIME`
- **대용량:** 텍스트는 `TEXT`/`LONGTEXT`, 바이너리는 `BLOB`/`LONGBLOB`

### 주의사항
- DDL은 실행 후 **복구 불가능**
- 테이블/칼럼 이름은 snake_case 권장
- AUTO_INCREMENT는 PRIMARY KEY 필수
- 적절한 데이터 타입 선택으로 성능 최적화
