# 관계형 DBMS

## 1. 관계형 DBMS 개요

### 1.1 정의

**Relational Database Management System (RDBMS)**

데이터를 **테이블(Table)** 구조로 저장하고 관리하는 데이터베이스 관리 시스템

### 1.2 특징

**장점:**
- 편리한 유지 보수
- 대용량 데이터 관리 용이
- 데이터 무결성 보장

**단점:**
- 시스템 자원을 많이 차지
- 구조가 복잡해질수록 성능 최적화 필요

### 1.3 대표적인 관계형 DBMS

| DBMS | 특징 |
|------|------|
| Oracle | 오라클사, 가장 널리 사용, 상용 |
| MySQL | 오라클사, 오픈소스 기반, 무료/상용 버전 |
| MariaDB | MySQL 수정 버전, 오픈소스 |
| MS SQL Server | 마이크로소프트사, MS 생태계 호환 |
| PostgreSQL | 오픈소스, ORDBMS (객체 지향 특성) |

## 2. 테이블 (Table)

### 2.1 정의

데이터를 표의 형식으로 체계화하여 행과 열의 집합으로 구성한 것

**동의어:** 릴레이션 (Relation)

### 2.2 테이블 구성 요소

| 구성요소 | 설명 | 동의어 |
|----------|------|--------|
| 행 (Row) | 실제 저장된 데이터 | Record, Tuple |
| 열 (Column) | 데이터의 세부 특성 | Field, Attribute |

**예시:**
- 행: 홍길동이라는 사람 (데이터)
- 열: 이름, 나이, 주소 (특성)

### 2.3 테이블 관련 용어

**도메인 (Domain):**
- 특정 칼럼이 가질 수 있는 값의 범위
- 예: `성별` 칼럼 → `남`, `여` 중 하나

**차수 (Degree):**
- 테이블의 칼럼(열) 개수

**기수 (Cardinality):**
- 테이블의 로우(행) 개수

## 3. 키 (Key) 종류와 역할

### 3.1 키의 정의

테이블의 데이터(행, Row)를 고유하게(Unique) 식별할 수 있는 칼럼 또는 칼럼의 조합

### 3.2 키의 종류

```
슈퍼키 → 후보키 → 기본키
                ↘ 대체키
외래키 (별도)
```

### 3.3 슈퍼키 (Super Key)

**특징:**
- 각 데이터를 식별할 수 있는 칼럼 또는 칼럼의 집합
- 유일성 (Unique) 만족
- 두 개 이상의 칼럼 조합 가능

### 3.4 후보키 (Candidate Key)

**특징:**
- 최소한의 칼럼으로 구성된 슈퍼키
- 유일성 + 최소성 만족

### 3.5 기본키 (Primary Key)

**정의:**
- 후보키 중에서 DBA가 선택한 키
- 데이터 식별을 위한 식별자(Identifier) 역할

**특징:**
- NOT NULL: NULL 값 불가
- UNIQUE: 중복 값 불가

**기본키 종류:**

| 종류 | 설명 | 예시 |
|------|------|------|
| 자연키 (Natural Key) | 실제 존재하는 데이터 | 아이디, 이메일 |
| 인공키 (Artificial Key) | 인위적으로 추가한 키 | 주문번호, 고객번호 (Auto Increment) |

### 3.6 대체키 (Alternate Key)

**정의:**
- 후보키 중 기본키를 제외한 나머지 키
- 보조키라고도 함

### 3.7 키 선정 예시

**고객 테이블:**

| 고객번호 | 아이디 | 고객명 | 직업 | 휴대전화번호 | 주민등록번호 |
|----------|--------|--------|------|--------------|--------------|
| 1 | aaa | 제임스 | 자영업 | 010-1234-1234 | 851122-111111 |
| 2 | bbb | 에밀리 | 개발자 | 010-8282-8282 | 951017-222222 |

**슈퍼키:**
- 고객번호
- 아이디
- 휴대전화번호
- 주민등록번호
- 고객명 + 직업

**후보키:**
- 고객번호
- 아이디
- 휴대전화번호
- 주민등록번호

**기본키:**
- 고객번호 (선정 이유: NOT NULL 보장, 개인정보 이슈 없음)

### 3.8 외래키 (Foreign Key)

**정의:**
- 다른 테이블과 관계(Relationship)를 맺기 위한 키

**특징:**
- 다른 테이블의 기본키(PK) 또는 Unique 칼럼과 관계 맺음
- NULL 값 또는 중복 값 가능
- 참조하는 기본키의 값만 가질 수 있음

**부모/자식 테이블:**
- **부모 테이블:** 참조되는 테이블
- **자식 테이블:** 외래키를 가진 테이블

## 4. 참조 무결성

### 4.1 정의

외래키로 지정된 칼럼은 참조 중인 부모 테이블의 데이터만 가질 수 있음

### 4.2 참조 무결성 위배 상황

부모 테이블의 데이터가 삭제되거나 수정되는 경우 발생

### 4.3 참조 무결성 위배 방지 장치

**1) RESTRICT**
- 참조 무결성 위배 시 작업 취소

**2) CASCADE**
- 부모 테이블 데이터 수정/삭제 시 자식 테이블 데이터도 함께 수정/삭제

**3) SET NULL**
- 부모 테이블 데이터 수정/삭제 시 자식 테이블 데이터를 NULL로 설정

## 5. 데이터베이스 모델링

### 5.1 모델링 단계

```
개념적 모델링 → 논리적 모델링 → 물리적 모델링
```

### 5.2 개념적 모델링

**목적:** 데이터 간의 관계 정의

**도구:** ERD (Entity Relation Diagram)

**표기법:** 피터 첸 표기법 (Peter Chen Notation)

**ERD 기호:**
- 사각형: 엔티티 (Entity)
- 타원: 속성 (Attribute)
- 마름모: 관계 (Relationship)
- 밑줄: 주식별자 (Primary Identifier)

### 5.3 논리적 모델링

**목적:** 테이블과 칼럼 생성

**특징:** 읽기 쉽게 한글로 작성

### 5.4 물리적 모델링

**목적:** 실제 데이터베이스에 적용

**특징:** 테이블/칼럼명을 영문으로 작성

---

## 6. 핵심 요약

### 관계형 DBMS
- 테이블 구조로 데이터 관리
- 무결성 보장, 유지 보수 용이

### 테이블 구성
- 행 (Row): 데이터 (Record, Tuple)
- 열 (Column): 특성 (Field, Attribute)

### 키 체계
- 슈퍼키 (유일성) → 후보키 (유일성 + 최소성) → 기본키 (선택된 키)
- 외래키: 테이블 간 관계 설정

### 참조 무결성
- RESTRICT: 작업 취소
- CASCADE: 함께 수정/삭제
- SET NULL: NULL로 설정

### 모델링
- 개념적: ERD 작성
- 논리적: 한글 명명
- 물리적: 영문 명명
