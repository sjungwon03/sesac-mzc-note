# 내장 함수

## 1. 내장 함수 개요

### 1.1 내장 함수란?

MySQL에서 기본 제공하는 함수로, 문자열 처리, 숫자 계산, 날짜 계산, 조건 분기 등을 SQL 안에서 수행할 수 있게 해준다.

### 1.2 사용 목적

- 조회 결과를 원하는 형태로 가공
- 날짜나 숫자 계산 처리
- `NULL` 값 대체
- 조건에 따른 결과값 변환

## 2. 문자열 함수

### 2.1 CHAR_LENGTH와 LENGTH

| 함수 | 설명 |
| --- | --- |
| `CHAR_LENGTH(str)` | 문자열의 글자 수 반환 |
| `LENGTH(str)` | 문자열의 바이트 수 반환 |

```sql
SELECT CHAR_LENGTH('hello'), LENGTH('hello');
SELECT CHAR_LENGTH('안녕'), LENGTH('안녕');
```

UTF-8 기준 한글은 보통 한 글자가 3바이트이므로 글자 수와 바이트 수가 다를 수 있다.

### 2.2 CONCAT과 CONCAT_WS

| 함수 | 설명 |
| --- | --- |
| `CONCAT(str1, str2, ...)` | 문자열을 그대로 이어 붙임 |
| `CONCAT_WS(separator, str1, str2, ...)` | 구분자를 넣어 문자열을 이어 붙임 |

```sql
SELECT CONCAT('2026', '06', '02');
SELECT CONCAT_WS('-', '2026', '06', '02');
```

### 2.3 SUBSTRING

문자열의 일부를 추출한다.

```sql
SELECT SUBSTRING('대한민국만세', 3, 2);
```

결과는 `민국`이다.

**주의:** MySQL 문자열 위치는 1부터 시작한다.

## 3. 수학 함수

### 3.1 ROUND

지정한 위치까지 반올림한다.

```sql
SELECT ROUND(123.456, 2);
```

결과는 `123.46`이다.

### 3.2 TRUNCATE

지정한 위치 아래를 버린다.

```sql
SELECT TRUNCATE(123.456, 1);
```

결과는 `123.4`이다.

### 3.3 MOD

나머지 값을 구한다.

```sql
SELECT MOD(10, 3);
```

결과는 `1`이다.

## 4. 날짜 및 시간 함수

### 4.1 NOW와 SYSDATE

| 함수 | 설명 |
| --- | --- |
| `NOW()` | 쿼리 실행 시점의 날짜와 시간 반환 |
| `SYSDATE()` | 함수 호출 시점의 날짜와 시간 반환 |

```sql
SELECT NOW();
SELECT SYSDATE();
```

실무에서는 일반적으로 트랜잭션 정합성을 위해 `NOW()`를 사용하는 것이 좋다.

### 4.2 DATE_ADD와 DATE_SUB

날짜에 특정 기간을 더하거나 뺀다.

```sql
SELECT DATE_ADD('2026-06-02', INTERVAL 1 DAY);
SELECT DATE_SUB('2026-06-02', INTERVAL 1 MONTH);
```

사용 가능한 단위 예시:

| 단위 | 의미 |
| --- | --- |
| `YEAR` | 년 |
| `MONTH` | 월 |
| `DAY` | 일 |
| `HOUR` | 시 |
| `MINUTE` | 분 |
| `SECOND` | 초 |

### 4.3 DATEDIFF와 TIMEDIFF

| 함수 | 설명 |
| --- | --- |
| `DATEDIFF(date1, date2)` | 두 날짜의 일수 차이 반환 |
| `TIMEDIFF(time1, time2)` | 두 시간의 차이 반환 |

```sql
SELECT DATEDIFF('2026-06-05', '2026-06-02');
SELECT TIMEDIFF('15:30:00', '12:10:00');
```

## 5. 제어 흐름 함수

### 5.1 IF

조건이 참이면 두 번째 값, 거짓이면 세 번째 값을 반환한다.

```sql
SELECT IF(100 > 200, '참', '거짓');
```

### 5.2 IFNULL

첫 번째 값이 `NULL`이면 두 번째 값을 반환한다.

```sql
SELECT IFNULL(NULL, '대체값');
SELECT IFNULL('데이터', '대체값');
```

조회 결과에서 `NULL` 대신 기본값을 보여줄 때 자주 사용한다.

### 5.3 CASE

여러 조건을 순서대로 검사해 결과를 반환한다.

```sql
SELECT emp_name,
       salary,
       CASE
           WHEN salary >= 5000000 THEN '고급'
           WHEN salary >= 3000000 THEN '중급'
           ELSE '일반'
       END AS salary_grade
FROM employees;
```

## 6. 함수 사용 시 주의사항

### 6.1 WHERE 절에서 칼럼 가공 주의

인덱스가 걸린 칼럼을 함수로 감싸면 인덱스를 사용하기 어려울 수 있다.

```sql
-- 인덱스 사용 어려움
WHERE DATE(created_at) = '2026-06-02';

-- 인덱스 사용 가능성 높음
WHERE created_at >= '2026-06-02 00:00:00'
  AND created_at < '2026-06-03 00:00:00';
```

### 6.2 NULL 처리 주의

`NULL`은 일반 비교 연산자로 비교하지 않는다.

```sql
-- 잘못된 방식
WHERE deleted_at = NULL;

-- 올바른 방식
WHERE deleted_at IS NULL;
```

## 7. 핵심 정리

- 문자열 함수는 데이터 출력 형태를 가공할 때 사용한다.
- 수학 함수는 반올림, 절사, 나머지 계산 등에 사용한다.
- 날짜 함수는 기간 계산과 현재 시각 조회에 사용한다.
- `IFNULL`은 `NULL` 대체값 처리에 유용하다.
- `CASE`는 SQL 안에서 다중 조건 분기를 표현할 때 사용한다.
