# 데이터베이스 이해 학습 노트

## 1. 데이터베이스 개요

### 1.1 파일 시스템

**정의:**
- 데이터베이스 이전의 데이터 저장 방식
- 데이터를 파일 형식으로 저장, 관리

**단점:**
- 데이터 중복 저장으로 기억 장소 낭비
- 파일 공유 어려움
- 보안 조치 미흡
- 데이터 검색 명령어 부재

### 1.2 데이터베이스

**정의:**
- 여러 사용자나 응용 프로그램에 의해 공유되어 사용될 목적으로 통합, 구조화되어 저장, 관리되는 데이터들의 집합
- 데이터 저장 공간 자체를 의미

**특징:**

| 특징 | 설명 |
|------|------|
| 데이터 무결성 | 데이터는 항상 오류가 없는 상태 |
| 데이터 중복 최소화 | 동일 데이터의 중복 저장 방지 |
| 데이터 보안 | 허가된 사람만 접근 가능 |
| 데이터 안정성 | 손상되어도 원상 복구 가능 |

## 2. DBMS (Database Management System)

### 2.1 정의

데이터베이스를 관리하고 운영하기 위한 시스템 (소프트웨어)

사용자나 응용 프로그램이 DBMS가 관리하는 데이터에 동시 접속하여 데이터를 공유할 수 있음

### 2.2 주요 DBMS

- Oracle
- MySQL
- Microsoft SQL Server
- PostgreSQL
- MongoDB
- Redis
- MariaDB

## 3. DBMS 종류

### 3.1 계층형 DBMS

**구조:**
- 각 계층이 트리 형태
- 각 노드는 1:N 관계

**장점:**
- 구조가 단순함

**단점:**
- 구조 변경 어려움
- 접근 유연성 부족

### 3.2 망형 DBMS

**구조:**
- 각 노드가 그래프 형태
- 1:1, 1:N, N:M 관계 지원

**장점:**
- 다양한 관계 지원
- 효과적이고 빠른 데이터 조회

**단점:**
- 프로그래머가 모든 구조를 알아야 함
- 프로그램 작성 어려움

### 3.3 관계형 DBMS (RDBMS)

**구조:**
- 데이터를 테이블(Table)에 직관적으로 저장
- 각 행: 키(Key)를 포함한 데이터
- 각 열: 데이터의 속성(Attribute)

**특징:**
- DBMS 중 가장 많이 사용되는 주류

**장점:**
- 직관적인 구조
- 데이터 무결성 보장
- 유지 보수 용이

### 3.4 객체 지향형 DBMS (OODBMS)

**구조:**
- 정보를 객체(Object) 형태로 표현
- 객체 간 상속 관계로 계층적 구조 구현

**특징:**
- 객체 지향 프로그래밍과 호환

## 4. DBMS 종류 비교

| DBMS 종류 | 구조 | 관계 지원 | 장점 | 단점 |
|-----------|------|-----------|------|------|
| 계층형 | 트리 | 1:N | 단순함 | 유연성 부족 |
| 망형 | 그래프 | 1:1, 1:N, N:M | 빠른 조회 | 복잡함 |
| 관계형 | 테이블 | 다양함 | 직관적, 주류 | 자원 소모 |
| 객체지향형 | 객체 | 상속 | 계층 구조 | 복잡함 |

## 5. 파일 시스템 vs 데이터베이스

| 구분 | 파일 시스템 | 데이터베이스 |
|------|-----------|-------------|
| 데이터 중복 | 높음 | 최소화 |
| 데이터 공유 | 어려움 | 용이 |
| 보안 | 미흡 | 체계적 |
| 데이터 검색 | 제한적 | SQL 등 강력한 명령어 |
| 무결성 | 보장 안됨 | 보장됨 |
| 안정성 | 낮음 | 높음 (복구 가능) |

## 7. 핵심 요약

### 데이터베이스 정의
- 통합, 구조화된 데이터 집합
- 공유 목적으로 저장, 관리

### 데이터베이스 특징
- 무결성: 오류 없는 상태
- 중복 최소화: 공간 효율
- 보안: 접근 제어
- 안정성: 복구 가능

### DBMS 정의
- 데이터베이스 관리 시스템 (소프트웨어)
- 동시 접속 및 데이터 공유 지원

### DBMS 종류
- 계층형: 트리 구조, 단순하지만 유연성 부족
- 망형: 그래프 구조, 빠르지만 복잡
- 관계형: 테이블 구조, 현재 주류
- 객체지향형: 객체 구조, 계층 표현 용이

### DBMS 선택 기준
- 용도 및 규모
- 비용 (상용 vs 오픈소스)
- 성능 요구사항
- 개발 생태계
